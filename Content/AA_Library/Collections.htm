Collections

In Processing Mode

There are three entity types described below, which are used to support the creation and maintenance of collections.

  • Collection groups
  • Dynamic collections
  • Static collections
Note

Users must have access to the 'Maintain Collections' function in System Administration to be able to create collection groups. Refer to System Administration User Guide for details on how to create a collection group entity.

Understanding Collections

To view all entities that form a collection

  1. Collections are accessed by selecting the 'Entities' drop-down at the top of the library tree, and changing the selection to 'Collections'.

    Note

    Only collections that are included in the entity set to which a user group has access will be displayed. These settings are maintained in System Administration.

  2. Right click on the collection and select 'Entities'.

  3. This opens a new search view, listing all of the entities in the collection.

Dynamic Collections

Dynamic collection entities

Dynamic collections are the result of a defined search. This is similar to the F3 search functionality, where attribute values are used to find all entities that meet the selection criteria. For example, all entities with 'Shares' in their description name; or all entities with a 'time to be released by' = 2pm. Unlike the search, the dynamic collection remains constant each time pControl is opened. It can also be viewed and edited by other users who have access to the entities within the collection. Where a new entity is created which contains the attribute criteria specified in the search, the new entity will automatically form part of the dynamic collection.

The selection criteria for a dynamic collection can be changed or deleted, by opening the properties view and modifying the criteria, or deleting rows.

 

 To create a dynamic collection entity
  1. Right click on the collection group entity, and select New > Collection > Dynamic Collection.

  2. Enter a code and description for the Dynamic collection entity.

  3. Specify the selection criteria for the collection, by choosing an attribute group, attribute, and (optionally) values for that attribute. In this example, the dynamic collection consists of all entities where 'Data Source 1' is the primary data source type.

    Additional selection criteria can be specified by selecting Add.

  4. To see which entities meet the specified selection criteria, select Entities at the bottom of the window. This will open a new search tab in the same way as the F3 search, displaying the relevant entities, or displaying nothing if no entities meet the criteria.
  5. When all selection criteria is complete, select OK to save the changes and create the collection, or Cancel to exit without saving.

  6. The collection entity will now appear under the group.

 

Static Collections

Static collection entities

Static collections consist of a user chosen selection of up to 160 entities. Static collections are viewable and editable by other users, with the required security access. These collections will also remain from one pControl session to the next.

 

 To create a static collection entity
  1. Right click on the collection group entity, and select New > Collection > Static Collection
  2. Enter a code and description for the collection entity.

  3. Select the entities to form the collection by selecting the required entities and then clicking the arrow buttons to move them across to the 'selected' category.

  4. When all entities have been selected, select OK to save the changes and create the collection, or Cancel to exit without saving.

  5. The collection entity will now appear under the group.
