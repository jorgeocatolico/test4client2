Entity Types

Different entities are used for different purposes. Some can have processes performed, others, such as process templates, company entities, and company default entities are used to support processing.

 Company entities

The 'Company' entity is a unique entity in pControl, as it is at the highest level in the library tree, and it is used to house all other entities. No processes are run directly against company entities. This entity must be created within pControl System Administration, before any underlying entities can be created.

Company entities contain a 'Company Default' attribute which is a link to another entity on the library tree. This is where default attribute configuration is set. If any entities underneath the company entity in the library tree do not have these attributes set as part of their configuration, they will inherit the attribute configuration from the 'Company default' entity.

There are various tabs on the default company entity, these tabs are named according to the specific process the attributes relate to. Attributes for each specific process are explained within the respective dashboard.

 Processing Entities

'Processing entity' refers to any entity which can have a pControl process run against it, including those listed below.

Entity Type

Description

Benchmark

A benchmark entity is used to represent a market benchmark or a composite benchmark that is used in pControl mainly for indexed valuation; price validation or CPU validation purposes.
Class

A class entity is used to represent a class of units in a multi-class vehicle where a class may typically only have one investment into a pool which it shares with other unit classes.

Currency

Currency entities are used to hold FX rate information. Valuation 1 can be run on a currency entity to validate FX rates / calculate and store cross rates.

External Product

An external product entity is used to represent an external unitised product where the unit price, distribution categories, CPU rates (and values) etc. are notified by the product.
Pool

A pool entity is used to represent a pool of assets which are shared by many classes of units. Pool entities can strike notional prices, distribution rate etc. which are then used by the unit classes ‘investing’ into it to calculate their own prices, distribution values etc.
Portfolio

A portfolio entity is used to represent a portfolio of assets in pControl where the value of the fund, income items etc. are typically received from an external investment management system or third party provider.
Unitised Product

A product entity is used to represent a standard unitised vehicle or fund in pControl on which processing may be run (e.g. Unit Pricing, Distribution Calculation etc.).

To view the processes attached to an entity
  1. Select the entity on the library tree.

  2. The processes associated with the entity will display on the left, as shown, and the attributes will display to the right.

  3. To view the phases and attributes of a process, select the arrow next to the process name i.e. Cash Allocation.
