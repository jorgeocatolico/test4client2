Validation Checks

[FVC001] - Security Price Movement Check

 

Business Description

This validation checks if the movement in the price of the security since prior day has exceeded the tolerance.

Processing Detail

The validation is performed on the price from the primary price source, the U175: Security Prices worktable, and checks if the price has moved out of tolerance since prior day.

The check calculations are:

$Check\ Value\ =\ \large \frac{Current\ Primary\ Price }{Prior\ Primary\ Price\ -1}$

where:

$\small Current\ Primary\ Price\ =\ A50.Primary\ Price\ at\ T$

$\small Prior\ Primary\ Price\ =\ A50.Primary\ Price\ at\ T-1$

Entity Data Attributes Used
  • nil

Interdependencies
  • The SAM data is required from the A50: Security Asset Master Worktable

Prerequisite Functions
  • nil

Outputs
  • Process status messages report the status.
  • If the [FVC001] check breaches the tolerance set, an exception will be raised.
  • Results of this validation check are written to the A52 : Security Asset Master Daily Plan worktable

 
[FVC002] - Security Price Impact Check

 

Business Description

This validation checks if the movement in the market value of the security, from prior pricing point has had a material impact on the NAV or UOI of the fund which holds that asset.

Processing Detail

The validation runs for all funds on which the security is held and creates one validation output record per security per price group. For example, if the security is held on multiple funds linked to the same price group, one exception would be raised which lists each of the pools on which tolerance has been breached.

The check calculations are:

$NAV\ Impact\ =\ \Large \frac{ \frac{\left [\left ( SAM\ Primary\ Price\ -\ SAM\ PD\ Price \right ) \times\ Unit\ Holdings \times\ A03.pricing\ factor \right ] }{Holding\ FX\ Rate\ } }{NAV}$

or

$\small Per\ Unit\ Impact\ =\ \Large \frac{ \frac{\left [\left ( SAM\ Primary\ Price\ -\ SAM\ PD\ Price \right ) \times\ Unit\ Holdings \times\ A03.pricing\ factor \right ] }{Holding\ FX\ Rate\ } }{UOI}$

where

SAM is the A50: Security Asset Master record.

UOI is derived from the U108: Units on Issue worktable.

Entity Data Attributes Used
  • nil

Interdependencies
  • The SAM data is required from the A50: Security Asset Master Worktable

Prerequisite Functions
  • nil

Outputs
  • Process status messages report the status.
  • If the [FVC002] check breaches the tolerance set, an exception will be raised.
  • Results of this validation check are written to the A52 : Security Asset Master Daily Plan worktable

 
[FVC003] and [FVC003-PUI] - Unchanged Price Movement Check

 

Business Description

This validation checks if the security price has remained unchanged since the prior pricing point.

Processing Detail

If the market price date is equal to the current effective date, and the movement of the security is zero, the check will fail and an exception will be raised.

$\small \%NAV\ =\ \large \frac{ Shares\ held\ by\ fund\ \times\ Current\ Price}{Total\ Net\ NAV\ of\ the\ fund} \small \times 100$

Where:

Shares held = A03.Holding Cur as at T-1

Current Price = A50.Primary Price at T

Net NAV Amount = A01.Net NAV Amount as at T-1 (or latest U101.net_nav_amount value.)

 

Similarly the [FVC003-PUI] check tests the per unit impact to determine if an exception will be raised.

$\small PUI\ Impact\ =\ \large \frac{Market\ Value\ of\ the\ asset\ in\ the\ pool}{UOI\ of\ the\ pool}$

Where:

Market Value = the A03.value_cur_base as at T-1

UOI = U108.confirmed_units as at T-1

Note: Where one asset is held by many funds, the fund with the largest impact caused by this validation breach will be displayed in the exception message.

Entity Data Attributes Used
  • nil

Interdependencies
  • SAM data is required from the A50: Security Asset Master Worktable
  • U175: Security asset prices

Prerequisite Functions
  • nil

Outputs
  • Process status messages report the status
  • If the check fails, an exceptions will be raised on the Exceptions view, and will include '%NAV' and 'PUI Impact'.
  • Results of this validation check are written to the A52 : Security Asset Master Daily Plan worktable

[FVO001] - Prior Day Price Consistency Check

 

Business Description

[FVO002] - Prior Day Price Source Consistency Check

 

Business Description

This validation checks if the prior day price source is consistent with Price Source Agreements (PSA) and the pools or funds holding the asset.

Processing Detail

Where an exact match is not found, the check will fail and an exception will be raised.

The check calculations are:

$\small A50.price\ source <> A03.price\ source$

AND

$\small A50.price\ source <> PSA.price\ source$

[FVO006] and [FVO006-PUI] - Prior Day Price Vendor Comparison Impact Check PR-6596 - assigned to Michael McCarthy, ready for review

 

Business Description

This validation checks if the variance in prices received across price vendors on the prior day, has a material impact on the NAV or UOI, of any funds which hold the asset. The impact of the variance between Primary, Secondary and Tertiary prices are checked.

Processing Detail

Where an exact match is not found, i.e. where all of the following criteria are met, the check will fail and an exception will be raised.

The check calculations for [FVO006] are:

$\frac{ \left ( \frac{ \Large\ Largest\ Price\ Source\ Price\ Difference\ \times\ Current\ holdings\ \times\ Pricing\ Factor }{ \Large\ FX\ Rate\ Currency} \right ) }{ \normalsize Current\ NAV}$

The check calcuations for [FVO006-PUI] are:

$\frac{ \left ( \frac{ \Large\ Largest\ Price\ Source\ Price\ Difference\ \times\ Current\ holdings\ \times\ Pricing\ Factor }{ \Large\ FX\ Rate\ Currency} \right ) }{ \normalsize UOI}$

Where:

Largest Price Source Price Difference is the highest absolute value returned from:

$ \small Primary\ Price - Secondary\ Price$

$ \small Primary\ Price - Tertiary\ Price$

$\small Secondary\ Price - Tertiary\ Price$

  • nil
