Common tab

Attributes displayed on the Common tab of processing entities, such as the asset type, and currency, are used by one or more processes. The following table details the attributes of the common tab.

Attribute Tab

Attribute Field

Description

Common

Asset Type

Indicates whether the entity represents a hedge fund or mutual fund, for differences in asset transaction processing.

Base Currency

The default currency of an entity. This is a mandatory field when setting up a new entity. It is used for reporting in accounts views and in currency conversions.

Custodian?

If selected, the entity is recognised as a custodian, and can therefore trade certain assets on behalf of a target fund with a specific list of counterparties.

Transaction Type Template

This attribute defines all transaction types used for various processes which can be run against this entity.

Funds Processing

Investor can receive Income

Specifies if the entity can be an investor (either a registered or beneficial owner) of an external product and who can receive income.

Life or Pension

Specifies the fund classification to be executed, either life or pension.

Default Settlement Type

Stipulates whether a unitholder will be paid their income in cash or reinvestment units if they have not notified their reinvestment preference (and the fund is an income fund).

Nominee Account

The nominee account being used by the company that the originator for FUM is in.

Allow beneficial owner display

Enables the management and display of the 'Beneficial Owner' for unitholders view; where pControl may be acting as a type of Nominee such as for 'Deal Consolidator' or 'Box Management' entities.

Fund's Sector

The sector of the fund i.e. fixed interest, property.

Fund id

The internal system id of the fund.

Express distribution value per unit as

Depicts how to display values.

Income or Accumulation

Type of fund.

Foreign Exchange

FX Rate Type

The default FX Rate Type is used by an entity when FX Rate Type is not configured for a particular process. The drop-down list includes 0800 or End of Day.

Note: If FX Rate Type is not configured for an entity, pControl will use 'End of Day' as the default FX Rate Type.

FX Price Type

The default FX Price Type used in a particular process by an entity. It is a mandatory field when setting up a new entity. The drop-down list includes Buy, Mid and Sell FX Rate.

Cash Account

Default Cash Account

The default cash account selected from this drop-down menu will be included as part of the cash balances in a business process (i.e. Daily Pricing or Cash Allocation) for an entity. It is a mandatory field when setting up a new entity.

Cash Account Category Code

When 'Load Cash Balance as Holding' in 'Data loads' is selected on the Daily Pricing process, this attribute becomes a mandatory field. The drop-down list contains the values from the U350: Classification Hierarchy.

Cash Account Instrument Group and Type

When 'Load Cash Balance as Holding' in 'Data loads' is selected on the Daily Pricing process, this attribute becomes a mandatory field. The drop-down list contains the values from U350: Classification Hierarchy.

Financials

Jurisdiction

A free text field to record the legal jurisdiction of this entity. This field is used for information purposes only. It is not linked to any rules.

Distribution Frequency

The validation to ensure the correct date is used for the distribution is based on the dates allocated in the Distribution Calendar attribute. This field is for information only.

Entity Tax Type

This attribute is used in L275: Tax Rate table and L380: Category split table.

Published MER Rate

This attribute stores the information on published Management Expense Ratio. For further information, refer to the pControl User Guide - Management Fee Rebate and Invoicing.

Base Class for comparative / Class for nil expense

If selected, this entity will not receive any apportioning of expenses during a distribution process.

Financial Year End Day

The day of the month the financial year ends for that entity.

Financial Year End Month

The month the financial year will end in for that entity.

Tax Year End Day

The day of the month the tax year ends for that entity.

Tax Year End Month

The month the tax year will end in for that entity.

Country of Domicile

This is a mandatory field when setting up a new entity.

Legal Structure

This is a mandatory field when setting up a new entity.

 

 Pricing Schedule (SLA) attributes

'Pricing Schedule' attributes are also known as SLA attributes. They are used for reporting the red / amber / green status of a process on the Process Summary view.

Note

The following example shows the 'Pricing Schedule' attributes for a daily pricing process, however the same attributes appear under most process tabs.

Attribute

Description
Price Release Location

The geographic location where the SLA time applies to.

E.g. if Daily Pricing is processed in India for an Australian fund, this attribute will be set to an Australian city location.
Use Effective Date If selected, the SLA will be calculated using the effective date of the process plus the offset.
Price to be Released By

The time a process needs to be completed by in order to meet its SLA.

Note: This is the local time at the 'Price Release Location'.

Price Release Offset Days

If the 'Use Effective Date' checkbox is selected, the SLA time will be calculated using the date and time the process is run + offset days.

E.g. If a fund is priced on T+2 basis, this attribute will be set to 2.

Data to be Loaded By

The time data needs to be received into pControl in order to meet its SLA.

Data Load Offset Days

If the data for an entity's process is receivedndays after T, (e.g. 2 days after T). This attribute will be set to 2

Price Run Type

The process type used to calculate a price or other output. This is especially useful for entities which have several processes run for them each day, e.g. Expected Pricing, Daily Pricing, Distribution etc.

The value entered into the price run type field will be used in the output of the process to identify the different processes.

Deferral Period (days)

The number entered here is added to the effective date for use in export worktables, such as the U141.Deferred Effective Date field.

E.g. If this attribute is set to 3, the above field on the U141 will be populated with a calculated date, i.e. effective date + 3.

Pricing Notes

The text entered will be displayed on all price records run for this fund.
