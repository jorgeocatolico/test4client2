Auto Clearance

Validation exceptions can be configured to be auto cleared if they meet certain pre-defined criteria. For example, if the movement of an asset price has triggered an exception, and auto clearance is configured for this check and fund; pControl is able to verify the asset price against a third part data source, and then auto clear the exception. This allows Funds Accountants to focus on the investigation of other more complex exceptions, ultimately reducing the time it takes to validate the NAV.

Auto cleared exceptions can be identified with the status of 'Auto Cleared'; They also have a pre-defined comment to indicate the method / reason for the auto clearance.

Configuring Auto Clearance entities

An 'Auto Clearance' entity is required for each rules file, and is used to house the auto clearance criteria.

In this example, AUTO_CLEAR_ASSET_PRICE contains the rules file, ACR001, which has been configured to compare the asset prices of U100: Investment Holdings to a third party pricing source. If the prices are within tolerance, any exceptions raised by the validation checks listed in the L600: Auto Clearance Validations worktable will be auto cleared.

Attributes of the Auto Clearance entity

Attribute Description

Rules File

The name of the rules file used to execute the auto clearance on any exceptions raised against validation checks listed in the L600: Auto Clearance Validations worktable.

Categories

Auto clearance will only be attempted on assets with the category code listed in the right hand column. I.e.: In this example, only 'Fixed Interest' and 'Equity' assets are eligible for auto clearance.

Price Comparison Rounding

Where the auto clear rule is comparing two different values, e.g. U100.Asset price and U175.Asset price, both values are first rounded to the number of decimal places shown in this attribute before they are compared.

Configuring Auto Clearance on a Fund

Auto clearance rules are turned on or off for each fund using the following attributes:

  • 'Auto Clearance of Exceptions allowed' check box - If this is selected, all auto clearance rules will be applied to any exceptions raised on applicable validation checks.


Auto Clear to State

Description

Auto Cleared

If selected, exceptions on any applicable validation checks, will be auto cleared, meaning, the user does not need to review the exception.

Proposed Auto Cleared

Exceptions on the applicable validation checks will be auto cleared to this state, and will then need to be reviewed and cleared by a user.

Note: If this attribute is set to blank, the value on the L600: Auto Clearance Validations worktable will be used.

Note: If the value selected in this attribute is different to the 'Auto Clear End State' selected on the L600: Auto Clearance Validations worktable, for a specified validation check, the lower value will be used. E.g. If the above attribute is set to 'Auto Cleared', and the L600.Auto Clear End State is set to 'Proposed Auto Clear' for HLD003 validation check, all HLD003 validation exceptions, will be transitioned to 'Proposed Auto Clear' and will require user review.

Note
The above example shows a pool fund, however these attributes are available under the 'Common' tab of all processing entities.

 

 Configuring Auto Clearance worktables

L600: Auto Clearance Validations

In this example, the auto clearance entity, AUTO_CLEAR_ASSET_PRICE is used to auto clear any exceptions raised against the validation checks listed in the L600: Auto Clearance Validations worktable:

Column Description

Validation Entity Code

The validation to be auto cleared.

Note: Validation entity codes can be found on the validation entity.

Auto Clearance Entity Code

Unique code of the auto clearance entity containing the rules file to execute the auto clearance check.

Sort Order

This numerical order determines the ascending order the auto clearances are executed in.

This allows the validation code that usually generates the most exceptions to be listed first (lowest number), so as to auto clear the largest number of exceptions first.

Action on Unknown

A drop down list of rules files, which contain an action to be taken if the auto clearance is not able to execute successfully.

For example, if an auto clearance check compares the price of an asset in the U100: Investment Holdings worktable, against the U175: Security Prices worktable, and there is no U175 record for a specific asset, the check cannot be completed, Therefore, the action listed in this field, such as make a request to Bloomberg for this particular asset price will be performed.

Autoclear End State

Auto cleared exceptions are transitioned to the lower of either the state specified in this field, or the state in the 'Auto Clear to State' attribute on the fund entity.

For example, the end state for HLD113 validation check is 'Auto Cleared'. If a fund entity is configured to transition all auto cleared exceptions to 'Proposed Auto Cleared', the auto clearance rule for HLD113 will only transition the exception to 'Proposed Auto Cleared'

Stop if Unknown

If this check box is selected, and the required data is not available to perform the auto clearance rules check specified in this row, all subsequent auto clearance checks will be stopped.

If not selected, each auto clearance check will be executed in order, regardless of whether the required data is available or not.

Active

Only rows with this checkbox selected will have auto clearance performed on the validation check in this row.

 

L610: Auto Clearance Valid Reason Codes

Auto cleared exceptions can be identified with the status of 'Auto Cleared', and they have a pre-defined comment to indicate the method / reason for the auto clearance. These comments are configured in the L610: Auto Clearance Valid Reason Codes worktable.

Column Description

Auto Clearance Entity Code

Unique code of the auto clearance entity containing the rules file to execute the auto clearance check.

Reason Code

List of valid reason codes for an auto clearance rule. When an auto clearance rule is run, it will clear the exception with the related fixed comment, where the underlying data has one of these reason codes.

Fixed Comment

The text to be used in the comments field on an auto cleared exception.

Manual

If this field is selected, and a user manually clears an exception, the fixed comment on this row will be used as the comment on the cleared exception, and the reason code will also be saved to the relevant worktable, e.g. U175 against the asset code which incurred the exception. This reason code, (now saved to a corresponding worktable, e.g. U175) can then be used by the auto clearance rule to clear any further exceptions on other funds for the same validation check, same asset, same effective date, and same price run type.

 

Auto Clearance Rules

The Auto Clearance Rules, run at the end of a NAV process, use predefined criteria to auto clear specified exceptions on funds configured for auto clearance.

The following drop downs contain the rules and criteria used during auto clearance.

ACR001: Asset Price Match

 

Business Description

Auto Clearance rule ACR001 can be used to auto clear validation exceptions caused by an asset's price, or price movement, if that price can be independently verified.

Allowable Validation checks

Auto Clearance Rule ACR001 can be applied to any or all of the following validation checks.

  • [HLD003] - Zero Market Price Check
  • [HLD012] - Zero Market Price Movement
  • [HLD051] - Security Price to Benchmark Movement
  • [HLD113] - Asset Price Movement Check

 

Data Requirements

[A03]: Holding Validation worktable

  • Asset Code
  • Market Price Date
  • Price Local Cur

Note: [EX210] - Export Holdings Summary function must be selected on the process template, to export the asset holdings from the U100 to the A03.

 

[U175]: Security Prices worktable

  • Asset Code
  • Effective Date
  • Price Run Type
  • Market Mid Price
  • Reason Code

 

Criteria for Auto Clearance

All of the following criteria must be met for an exception to be auto cleared:

  • The values in all fields of the below worktables must contain the exact same values:

A03:Holding Validation

U175: Security Prices

Asset Code

Asset Code

Market Price Date

Effective Date

Price Local Cur *

Market Mid Price *

Price Run Type

Security Price Run Type
  • U175.Reason Code field must contain a valid reason code
    • The reason code field can be populated either by pControl where it has been able to verify an asset's price against a market data provider's price, or manually, where a price investigation has occurred, and the price has been verified.

 

* Note: The U100 price local cur, and U175 market mid price are rounded to the number of decimal places specified on the Auto Clearance attribute, and then the prices are compared. For further details, refer to Configuring Auto Clearance entities

Action on Unknown

If the verified data required for auto clearance is not available, pControl will take the action specified in the L600: Auto Clearance Validations worktable.

Outputs

If the criteria for auto clearance is met, the exception will be auto cleared. The comment field will display the fixed comment text from the L610: Auto Clearance Valid Reason Codes worktable, which contains the corresponding reason code as the U175 record, used in the asset clearing process.

 
ACR010: Recurring Exceptions PR-6484

 

Business Description

Auto Clearance rule ACR010 can be used to auto clear the 'same' exception for a defined number of days, where the exception has been cleared for a specific reason.

Allowable Validation checks

Auto Clearance Rule ACR010 can be applied to any or all of the following validation checks.

  • [HLD003] - Zero Market Price Check
  • [HLD012] - Zero Market Price Movement
  • [[HLD013] - Market Price Date Check

 

Data Requirements

[A03]: Holding Validation worktable

  • Asset Code
  • Market Price Date
  • Price Local Cur

And:

  • Prior Day Exception Events

Criteria for Auto Clearance

All of the following criteria must be met for an exception to be auto cleared:

  • The exception is the "same" as the exception raised the previous day, that has been cleared. An exception is considered the "same" as the exception raised the previous day and cleared, where the following conditions are met:
    • Entity Code
    • Validation Code
    • Price Run Type
    • Test Item Entity e.g. Asset Code, Account Code
    • Test Item e.g. Asset Description
    • Test Item Transparency e.g. Currency
    • Test Item Group e.g. Category

    The following table defines "same" for each validation type:
  Conditions for Inheritance

[HLD003] - Zero Market Price Check
  • The Validation Code matches prior day
  • Asset Code matches prior day
  • Price of the asset is the same

[HLD012] - Zero Market Price Movement
  • The Validation Code matches prior day
  • Asset Code matches prior day
  • Current Market Price Date matches prior day
  • Asset Price remains unchanged.

[HLD013] - Market Price Date Check
  • The Current Market Price Date matches prior day
  • Asset prices remains unchanged
  • The Description matches prior day. - Carmel to ask Theresa what this means
  • The exception was cleared (validated), and a fixed reason code, as defined in the L610: Auto Clearance Valid Reason Codes worktable, was selected during validation.
  • The exception has not yet reached the maximum number of days allowed to auto clear as defined in the L610: Auto Clearance Valid Reason Codes.Recurring Days field.

 

Action on Unknown

If the criteria for auto clearance is not met, the exceptions are not cleared.

Outputs

If the criteria for auto clearance is met, the exception will be auto cleared. The comment field will display the fixed comment text from the L610: Auto Clearance Valid Reason Codes worktable.
