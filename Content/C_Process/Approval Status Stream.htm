Exceptions in Pricing

Exceptions have been raised by one or more of the processing phases which need to be overridden before the control record can be approved.

Overridden

N/A

When all exception events associated to the control record on the processing day have been overridden by a user, pControl’s business rules will move the status to ‘Overridden’ automatically.

Defaults to current date and time taken from entity attribute ‘to be released by’

Mandatory

Overridden with L2 Exceptions

N/A

When all exception events associated to the control record on the processing day have been overridden by a user, pControl’s business rules will move the status to ‘Overridden with L2 Exceptions’, where L2 exceptions were encountered (as defined on the L200 Tolerance worktable).

Defaults to current date and time taken from entity attribute ‘to be released by’

Mandatory

Overridden with L3 Exceptions

N/A

Defaults to current date and time taken from entity attribute ‘to be released by’

Mandatory

Overridden with L2 and L3 Exceptions

N/A

When all exception events associated to the control record on the processing day have been overridden by a user, pControl’s business rules will move the status to ‘Overridden with L2 and L3 Exceptions’ where both L2 and L3 exceptions were encountered (as defined in the L200 Tolerance worktable).

Defaults to current date and time taken from entity attribute ‘to be released by’

Unapproved

The status used to indicate that there were no exceptions found during the processing but the control record is yet to be approved

Control records that are ‘Unapproved’ will not be available for dependent control records (valuations/prices) to use.

Note: If an ‘Authorisation Level’ for approval has been configured in the entity attributes, the same authorisation level will need to be specified on the status stream transition alongside the user group which has permission to make this transition.

A user with the correct security access will be able to make this transition to approve a control record via the ‘Fund Sign Off’ view such that it can be released and any dependent control records (valuations/prices) can continue processing.

For distribution calculations, pControl checks that the Distribution Declared Control Record value equals the value in the Distribution Declared Tables.

If the validation had passed and Distribution Declared Control Record Value equals the value in the Distribution Declared Tables – then the record would be transitioned to Approved.

Recalculating

N/A

For distribution calculations when a user transitions to Pending Approval, pControl checks that the Distribution Declared Control record value equals the value in the Distribution Declared Tables.

If the validation fails pControl will further transition the state to ‘Recalculating’ – this state displays for a very short period as it then transitions to ‘Unapproved Recalculated’.

When this occurs the values in the pControl Distribution Declared table are updated and the values will agree to the Control record.

Released in Error

This state is used to signify that a control record has been released however it needs to be re-run.

From a status stream perspective, this state is an end state however from a business perspective, it is only an interim state where a released record must transition to if re-run is required. The process must be re-executed when control records are in this state.

Note: The behaviour on re-run is determined by the configuration of the ‘Mark as Approved on re-run’ default attribute on the Company entity. For further detail please refer to section 6.5 Default Processing Attributes.
