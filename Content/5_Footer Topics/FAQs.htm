FAQs

Process status is 'Waiting'. What is my process waiting for?

When a process is in a ‘Waiting’ state in the ‘Data Load’ phase, the process is waiting for one or both of the following:

  • Data (records) to arrive in the import worktables which have been configured as mandatory;
  • A control record (i.e. price, distribution rate etc.) for one or more of the entity’s underlying investments to be approved. Once this dependent process is completed and approved, the 'waiting' process will then continue on.
Note

In the Unit Pricing module, a manual prerequisite checklist can be configured as part of the unit pricing process. Once selected it becomes a mandatory part of the process, the unit pricing process will refer to that checklist in the P170: Prerequisite Streams worktable during different process phases. The process will enter a ‘Wait’ state until such time as the checklist has been completed.

To determine exactly what an entity is waiting for, there are two methods:

  • Processing Status Messages
    • Refer to the status messages (Ctrl + S) in the Processing Module; or the 'Status Messages' on the Price Control Summary view.
    • Drill down to the lowest level of the messages, this will indicate the process phase and function which are in the 'wait' state.
    • Refer to the relevant user guide for that function to determine the data requirement for that function. It is most likely the worktable used as the input source for that function has been set to 'Mandatory', and that worktable does not yet have today's data record. Once the record is loaded to that worktable, the process will continue.
  • Data Load Monitor – the data load monitor is a pControl worktable [P183] which is used by processing to record all of the ‘wait events’ that the process for a fund is currently waiting for.

 

 Why is my process in a state of 'Exception'

A process is usually in state of 'Exception' because a validation check has breached it's tolerance. The Exception must be investigated, and either overridden, or the underlying data corrected to move past the exception and allow the process to continue.

To find the exception, there are two methods:

  • Processing Status Messages
    • Refer to the status messages (Ctrl + S) in the Processing Module; or the 'Status Messages' on the Price Control Summary view.
    • Drill down to the lowest level of the message in the exception state, this will indicate the process phase and function where the exception occurred, as well as the reason for the exception.
    • The L200: Tolerance record for this validation check will also indicate why the exception occurred.
    • The exception can either be overridden on the 'All Prices' (processing mode) screen or the 'Price Control Summary' view (web mode). - Right click on the exception and override the exception. Refer to Processing Volume III guide on the PDF Downloads page for further detail on overriding exceptions.
  • Control Record View
    • Navigate to the 'All Prices' page (Processing Mode) or 'Price Control Summary' view (Web mode).
    • Select the control record in a state of 'Exception'; The second panel will open.
    • Select the 'Exceptions' tab in the second panel.
    • Locate the exception and investigate. Then either override the exception by right clicking and selecting 'override'; Or refer back to the L200: Tolerance worktable to determine if the tolerance values are correct; Or check the worktable data used as an input to the process to determine if it is correct.
Note

Some validation checks in the 'Post Processing Validation' phase, require one or more sources of data in order to perform the validation check. If those data values, or prerequisite process values are not loaded into the process during the 'Data Load' phase, the validation check may produce erroneous results or no results at all.

 

 Why is my process in a state of 'Assistance'?

A process will be in a state of 'Assistance' because an error has occurred, or the process has failed.

To find the source of the error, refer to the process status messages:

  • Navigate to Ctrl + S in the Processing Mode; or the 'Status Messages' on the Price Control Summary view.
  • Drill down to the lowest level of the message in the assistance state, this will indicate the process phase and function where the assistance occurred. The message displayed will indicate the source of assistance.
