Cash Allocation Functions

Calculate After Positions / Weights - NOT IN pCONTROL

Processing Phase

Post Processing

Description

This function calculates the after positions / weights for the entity, i.e., the % actual allocation that the fund will have into each of its investments if the proposed orders calculated in a preceding function were executed.

The calculation performed is as follows:

  • For each investment, the starting investment value is added to the proposed order to generate the proposed / after investment value
  • For each investment, the proposed / after investment value is divided by the total investment value of the fund to calculate the proposed/after weight of the investment and rounded to 4 decimal point.

All information on these calculations and tests is written out as status messages.

Function type

Process Template

Entity data attributes used

None

Tolerance type

No tolerance required

Pre-requisite functions

The following functions are optional pre-requisites but will impact the starting positions calculated by this function:

  • Calculate Liquidity Balance (Cash to Allocate)
  • Calculate Starting Positions/Weights
  • Calculate Proposed Orders

Outputs

 After positions / weights output to the status messages

 

 Write Proposed Orders - NOT IN pCONTROL

Processing Phase

Post Processing

Description

This function creates the Proposed Orders records in pControl with one of the following statuses:

  • Exception – if the ‘Run Compliance Rules’ option has been selected (see below for function description) and the proposed order has failed or breached the compliance rules that have been set for the fund, the proposed order is put into a state of ‘Exception’
  • Approved – when ‘Automatically Approve Proposed Orders’ attribute is selected, AND the ‘Automatically Approve Proposed Orders to State’ is set to ‘Approved’, the proposed orders are created with Approved state (if compliance rules have not been executed or have passed – see below), physical orders will also be created in pControl.
  • Calculated – when ‘Automatically Approve Proposed Orders’ attribute is selected, AND the ‘Automatically Approve Proposed Orders to State’ is set to ‘Calculated’, the Proposed Orders are created in a Calculated state (if compliance rules have not been executed or have passed – see below).
  • Calculated – Warnings – when ‘Automatically Approve Proposed Order’ is selected AND the ‘Automatically Approve Proposed Orders to State’ is set to ‘Calculated’ together with the event lists configured on the ‘issue Warning on these Events, the proposed orders will be created with a status of Calculated – Warnings if any of the events have been breached. For example, if the Issue Warning attribute has Min / Max Breach listed and the original ideal order breached the minimum or maximum transaction size, then the proposed order will be created with a ‘Calculated-Warning’ status.
  • Proposed – when ‘Automatically Approve Proposed Order’ is not selected, the proposed order will be created with a status of Proposed if none of the events listed in the ‘Issue Warning on these Events’ attribute have been breached.
  • Proposed – Warnings – when ‘Automatically Approve Proposed Order’ is not selected or the ‘Do not Automatically Approve on Warnings’ is selected together with the event lists configured on the ‘issue Warning on these Events, the proposed orders will be created with a status of Proposed – Warnings if any of the events have been breached. For example, if the Issue Warning attribute has Min / Max Breach listed and the original ideal order breached the minimum or maximum transaction size, then the proposed order will be created with a ‘Proposed-Warning’ status.

Note/s: For details on the fields on a Proposed Order refer to Appendix A – Proposed Order Record Definition.

For full explanation of proposed order states and transitions, refer to Section 5.3.2 Proposed Order Status Stream.

Function type

 Process Template

Entity data attributes used

 

Automatically Approve Proposed Orders

 

  • If selected, proposed orders will be automatically approved to a state which is specified in the Automatically Approve Proposed Orders to State.
  • When Automatically Approve Proposed Orders to State = Approved. The proposed orders will be created in Approved state and
    • If ‘Automatically Approve Orders’ is selected – Real orders will also be created in approved state or
    • If ‘Automatically Approve Orders’ is not selected – Real orders will also be created in unapproved state.
  • When Automatically Approve Proposed Orders to State = Calculated. The proposed orders will be created in Calculated state
  • If not selected, proposed orders will be created in Unapproved state
  • Do Not Automatically Approve on Warnings
  • If ‘Automatically Approve Proposed Orders’ is not selected and this attribute is selected, the proposed orders that have warnings on them will not automatically be approved. This attribute should be used in conjunction with the warning types listed in the ‘Issue Warning on these Events’ attribute as well as ‘Do Not Automatically Approve on Warnings (with dropdown list).
  • When Do Not Automatically Approve on Warnings (with dropdown list) = All Proposed Orders, none of the proposed orders will be approved if there are warnings on any other proposed orders.
  • When Do Not Automatically Approve on Warnings (with dropdown list) = Single Proposed Order, then the proposed order that has a warning will not be automatically approved, but other proposed orders (that do not have warnings) will be approved.
  • If ‘Automatically Approve Proposed Orders’ is not selected and this attribute is also not selected, proposed orders will be created in unapproved state.

Note: The Liquidity proposed order will always be created in the approved state regardless of whether proposed orders have been set to automatically approve.

Tolerance type

 No tolerance required

Pre-requisite functions

 Calculate Proposed Orders

Outputs

 Creation of proposed / physical orders depending on the attribute settings on the fund and the results of compliance processing.
