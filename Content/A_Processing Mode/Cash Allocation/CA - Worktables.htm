CA - Worktables

In Processing Mode

 

The main worktables referenced by Cash Allocation are listed below.

 

Import Worktables

Import worktables are used by the process to capture any data required during the cash allocation for an entity on a given processing day. Data is typically loaded into these worktables automatically using pControl Data Manager. Alternatively, data can be manually entered into these worktables through processing mode.

Import Worktable

Description

U101: External Prices

Holds any prices that have been imported to pControl from another system / location.

This worktable is used when pControl is not being used for the calculation of unit prices. The prices are an input into proposed order calculations.

U108: Units on Issue

Holds the units on issue of an entity.

U110: Cashflows and Orders

Holds details of cashflows from external registries or transfer agencies, or direct investments from investors. This data is used by the ‘Create Cashflow’ phase of processing or the ‘Create Registry Orders’ general process for the creation of registry orders which simulate the cashflow into the top-level or receiving funds.
U305: Cash Allocation - Liquidity Adjustments Holds adjustments that need to be made to the liquidity balance of an entity over a specified time period. Usually updated manually. Data for this buffer can be loaded from an external source automatically if required.Link to Load Liquidity Adjustments - in Templates
U306: Cash Allocation Liabilities Holds liability values of an entity for a specific date. Three different types of liability can be loaded into this worktable; close of business (CoB), intra-day, and manual adjustment. Link to Load COB / Intra-Day / Manual Adjustment Liabilities - in templates
U498: Investment Parameter Update Holds the allocation criteria, (target weights, tolerances and bounds) used by the Cash Allocation process to calculate how much cash to allocate to underlying assets. Link to Loading Allocation Parameters ALPs - in templates

 

 Static Worktables

Static worktables are used to hold data that does not change on a regular basis such as mappings and tolerances set for particular calculations.

Static Worktable

Description

L250: Transaction Flow Configuration

Records configuration relating to transaction types. Each transaction type can be configured for auto confirmation, auto cash settlement after approval, auto cash settlement after confirmation, and for the recording of a cash settle date offset.

Link to 5.6 what happens at the completion of the cash allocation process

The transaction types that are to be grouped must first be specified in the static data worktable, L250: Transaction Flow Configuration and selecting the 'Group in Cash Allocation' column.

Link to Calculate Proposed orders (template)

L260: Cash Allocation - Default Option Mappings

Maintains the relationships between a 'Default Option' entity and the set of investment entities. Appendix E - Default/ Balancing Options

 

 pControl Worktables

pControl worktables are usually read-only worktables used to hold the results of processing before being used in follow on processing, or for reporting within pControl.

pControl Worktable

Description

P500: Order Ids

Offset orders for the ‘default options’ are summed for each asset class, and one real order is created. This order is stored in this worktable.

For further info on P500 refer to UOM User Guide

P510: Message Interface

Used to monitor messages (i.e. relating to orders placed, approved, confirmed etc.).

This worktable represents both inbound and outbound messages, however, these are mutually exclusive, e.g. one row is required for inbound and one for outbound message types. Two status streams are used to display the inbound and outbound messages in the attention status monitor (ASM).

 

 Export Worktables

Export worktables are read-only worktables used to hold the results of processing. These worktables facilitate the export of any information that may be required for analysis or reporting outside of pControl.

pControl Worktable

Description

U510: Transaction Export

Used to record the details of transactions generated (i.e. orders generated from cash allocation processing). This buffer may be used to extract transaction details for ongoing processing (outside of pControl).
