Headings within drop-downs

Headings can either be h2.indropdown style or h3.

Always leave one p line free at the bottom of the drop down. This is because when tables or note/example boxes are used, or you use a drop down inside the drop down, the line becomes double lined and renders weirdly.

 

 Use of CAPS

Minimise the use of capitals where possible.

 Notes, Examples, Tips and Warnings

Divs are used for the boxes containing notes, examples, tips and warnings.

There are three note box options.

Note

A regular note box that looks like this using div.note_content.

2 paragraph lines.

Note

A note box that is short in width to cater for the mini TOC using div.note_content_in_list.

2 paragraph lines.

 

The final example is shown here in the list describing how to create a note box.

  1. Write the text first. With Note on one line followed by the text in the second line.

  2. Select the word "Note". Use the <Tab> key or the style window to select "Note Label". The word Note should now display in a purple box.
  3. Select the second line of text i.e. the note content. Use the <Tab> key or the style window to select the required note box style (div.note_content / div.note_content_in_list / div.note_content_short_width).

  4. Return to the Note Label and right click on the p to the left. Select Unbind to remove the p.

    Note

    If the p is not removed from the Note label, the label displays incorrectly on build.

 Image settings

There are two main styles associated with images:

  • For screenshots or images that require a zoom, the zoom4detail style is used. The zoom4detail style is applied to the p not the img. Used often when we are introducing a dashboard as we can insert a large size image. See Portfolio Management for examples.
  • For smaller images with white space (i.e. partial screen) or icons and images that fit within the text, a grey_border is used. Select the image and apply grey_border form styles. There are examples of this in CAR Process Phases and Functions.
  • We zoom out of pControl to 80% before taking a screenshot.

 

 Equations

Insert Menu > Equation

We use https://www.codecogs.com/latex/eqneditor.php as the initial editor, then transfer from there into flare. The reason we use it is because it has the maths symbols that we sometimes use.

For example, by typing the following into equation editor:

Ideal\ Order\ =\ \left (TW\ \times Total\ Investment\ Value\ of\ the\ fund \right )\ -\ Starting\ value\ of\ investment\ in\ the\ asset

Then inserting the same into Flare equation editor, renders:

$Ideal\ Order\ =\ \left (TW\ \times Total\ Investment\ Value\ of\ the\ fund \right )\ -\ Starting\ value\ of\ investment\ in\ the\ asset$

[REG010] - Generate Cashflow Records from Unit Movements

 

Business Description

This function compares the total units on issue loaded today against the total units on issue at the previous working day to calculate a unit inflow or outflow.

Processing Detail

This function is typically used by Life Companies for the generation of cashflow records from unit movements based on unit balances loaded into the U108: Units on Issue input worktable as sourced from Life Company policy administration systems.

Note: The unit inflow / outflow calculated here are used in later functions to calculate a cash amount cashflow record. Refer to [REG020] - Calculate Cash Amounts on Cashflow Records for further detail.

Entity Data Attributes Used
  • Cashflow Calculations
    • Note: The attributes are located on the policy administration system entity – they define the parameters of the originator of the entries. Multiple policy administration systems can be used to originate these entries. The parameters below can differ depending on the particular policy administration system.

Calculate Cash Amount
  • If this attribute is selected, pControl will calculate a cash amount and populate the ‘Cash Amount’ field together with the field ‘Calculate Cash Amount’ selected in the U110: Cashflows and Orders worktable.
    •  The cash amount will be calculated based on the unit movements between the two working days, X and the latest available price.
  • If this attribute is not selected, pControl will calculate the unit movements and populate the 'Unit Amount' field.

Positive Transaction Type

The transaction type that should be recorded against the record in the U110 worktable if the value is a positive amount (i.e. Application).

Negative Transaction Type

The transaction type that should be recorded against the record in the U110 worktable if the value is a negative amount (i.e. Redemption).

Price Type

The type of price i.e. Buy Price, Sell Price etc that should be recorded against the record in the U110 worktable.

Interdependencies
  • nil

Prerequisite Functions

In order for pControl to generate the cashflow records, the following additional data is required.

  • U108 Units on Issue – there should be two records present in the U108 worktable with the ‘Unit Mvt Status’ = ‘L’ (for loading) and ‘Calc Unit Mvt’ selected.

Note: In order to view ‘Unit Mvt Status’ and ‘Calc Unit Mvt’ fields, the U108 worktable must be in ‘Details’ view.

Outputs
  • U110: Cashflows and Orders records containing the unit amount (or with a zero unit amount in preparation for the [REG020] - Calculate Cash Amounts on Cashflow Records).
