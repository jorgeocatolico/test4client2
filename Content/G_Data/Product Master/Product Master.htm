Product Master

The Product Master worktable is a framework that enables the creation, maintenance, export and dissemination of product master data. This data can include information related investment firms, investment strategies and investment vehicles. The table allows for maintenance and aggregation of product master data as it relates to a firm's investment strategies, investment managers and related investable products that are maintained in pControl. The worktable has dual authorisation enabled so that modifications to the underlying data are correctly managed.

The Product Master worktable definition is maintained via the L080: Product Master, currently managed via the pControl System Administration module. Within pControl web, it is possible to add, delete and save investment strategies; as well as transitioning the record through the approval status stream. Additionally, within the worktable definition, product master fields can be defined to have editor type Worktable Tree Combo. Fields that have this editor type definition are maintain via the L081: Editor Options worktable in pControl web.

Product Master Worktable

Maintaining the Product Master worktable in System Administration

It is possible to modify the product master worktable via the pControl System Administration module by selecting Maintain > Dataviews, and finding the Product Master worktable in the options available. Once highlighted, select <Modify> to open the Data View window. Changes, such as making fields mandatory and optional, can be made that will be reflected in pControl Web, Data > Product Master.

Modifying Product Master Records in pControl Web

Linking a product master record to an entity

Each record in Product Master requires to be linked to an Investment Strategy entity in the library tree.

To create a new entity within the library tree:

  1. Select the required location in the library tree, right click and select 'Add'.

  2. Select a template group and Investment Strategy as the template type from the drop down menus.

  3. Select <Continue> to continue creating the entity, or <Cancel> to exit without saving.

  4. If continuing, a blank entity will open.

    Enter the Investment Strategy Code that will be used for the product master record. This will link the record with the entity.

  5. Select <Save> to save the new entity; or <Cancel> to exit without saving.

 

 To add, delete and save a record

When a record has been edited and saved, the change displays in red font while the record is pending approval. Once authorised, the original record is accessible using the red expand icon . Only the data that has been edited will be displayed in the second row. In the example below, the rows show that MG_BALANCED has been changed to MG_GROWTH. The full original record can be accessed via the ellipsis on the second row.

 

 Using the Properties view

Access a record's Properties window by selecting the ellipsis. All fields of the record can be edited by selecting the appropriate tab: Properties, Focus Tier and GICS Sector. The worktable leverages an editor type named Worktable Tree combo. This editor allows for smart filtering for multiple fields. For example, in the GICS sector tab, it is possible to select a sector at level 1, level 2 attributes become available and are subject to the level 1 choice. This is repeated through five levels.

 

 Validating records

When saving records, the validate function will check against settings stipulated in the System Administration set up. The following example shows Warnings displayed when a duplicate investment strategy code is used and where a mandatory field does not hold any data.

Selecting <Validate> will check all records that are currently in the 'In Edit' state. The validation process is triggered on selecting <Save> also.

To save and validate only one record, open the Properties view via the ellipsis and select <Save>.

 

 Using dual authorisation on product master records

When dual authorisation functionality is activated, records can be added, amended and deleted and a second person's authorisation will be required before the record can be transitioned to the approved state. Records with pending changes awaiting the second person's authorisation are highlighted in the worktable.

 

 

 Pending Changes on Product Master and Worktable Records

Records with pending changes can be viewed in the Pending Changes Dashboard which is accessible via Dashboard > Pending Changes Dashboard menu navigation. Further information is included under Pending Changes Dashboard.

Product Master records in a pending change state can be transitioned to the appropriate state in the status stream using the dual authorisation menu on the ellipsis, subject to security access. The authoriser or reviewer can also view additional details of the pending changes in the properties view by selecting the ellipsis.

The properties view highlights the changes have been made in blue text, and the pending changes panel provides the ability to view original and proposed values and the ability to transition the record to a new state.

 

 To search for a record

The Product Master view displays a maximum of 2000 records and has a context sensitive filter menu available to find one or multiple records. The menu can be accessed by the <More> option on the primary filter. Records can be filtered by name, i.e. entity name or investment strategy code, or filtered by investment firm or asset class.

Alternatively, the record can be found by searching for the entity on the primary filter as each product master record is linked to an entity. Select the <All Entities> drop down menu and type the name of the entity (also the Investment Strategy Code).
