U490: External Look-through

The U490: External Look-through worktable stores the investment path details or risk factors for the direct assets of a Cash and Rebalancing (CAR) process. These are recorded as a set of external child assets and their weights within the direct asset. This data is used by CAR for Classification Look-through when the look-through weights are externally supplied. As the child assets do not exist in pControl, the U485: External Child Asset data load and associated process, first needs to be run to create these assets.

Creating the external child assets

As previously noted, the external look-through worktable stores the investment path details or risk factors for the assets of a CAR process. These are recorded as a set of external child assets and their weight within the asset. However, these external child assets do not exist in pControl and need to be created as they need to be stored in the allocation parameters and allocation records as a pControl id. With the mandatory fields being code and description, these are stored in a node id table, similar to how the classification sectors are stored with a unique node type.

 Validations

The main validation is the pControl code is the sum of the Allocation Percent equals 100% for the bottom level of the look-through. However, multiple sets of look-through weights may be loaded for a pControl code. In these cases, the Classification Tag field must be populated. If a single set is presented, the Classification Tag field may be null.

 Validation conditions

Refer to L281: Validation Details for all validation criteria used by the Enrichment and Validation process.

 General Processes

When data is loaded into this worktable, the CA_LOAD_SECTORS general process must be run, to move records across to the required (java) database tables. When this process completes successfully, the status of the rows will be 'C' (Completed.)

If data is loaded to this worktable via the U495 Take On process, the CA_LOAD_SECTORS general process will be run automatically.
