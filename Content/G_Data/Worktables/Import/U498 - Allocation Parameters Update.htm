U498: Investment Parameters Update

 

Renamed to Investment Parameters following meeting with Andrew McLaren 16 Oct 18

Notes from Cash Allocation user guide; there's more on Allocation Parameters creating and viewing

Allows allocation parameters for an entity to be added or maintained in bulk. A general process 'Create ALPs from buffer' is then run to generate / modify the ALPs against the fund.

The CA_CREATE_ALPS general process in pControl is used to enable automated creation or modification of allocation parameters from the U498: Allocation Parameter Update worktable into pControl.

The U498: Allocation Parameter Update worktable can be updated manually or via automated file load. The CA_CREATE_ALPS process can then be run to load the ALPs from the worktable into pControl. The new allocation parameters can be viewed using the Function > Cash Allocation > View / Maintain > Allocation Parameters functionality.

To add ALPs to the worktable and load the ALPs from the worktable, the following steps should be followed.

  1. Navigate to the entity in the tree you wish to create an ALP for.

  2. Select Data > Buffer Data > Import > U498 Allocation Parameter Update.
  3. Select <Add> button from toolbar. A new blank row will be added to the worktable.
  4. Complete the relevant details for all required fields, for all of the ALPs that should be created.
  5. Select <Update> to save the record.
  6. Navigate to the general process entity, ‘CA_CREATE_ALPS’
  7. Right click on the entity, and select ‘Request Processing’

  8. Select ‘General’ in the process field, complete the processing/effective date fields, and select <Ok> to start the process / or select <Cancel> to cancel out of the process.

pControl will pick up all records matching that effective date from the import buffer and process them accordingly.

When the general process is run; pControl looks at the buffer data and using the following key (Originator Code + Counterparty Code + Asset Code + Start Date) determines how it should process the data.
