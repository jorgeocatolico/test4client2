U495: Classification Sectors

The U495: Classification Sector worktable allows one or more classification structures to be created, which classifies investment data by various criteria. These structures may be the same, or can be different to the investment or interfunding relationship structures. This worktable works in conjunction with the U496: Classification Allocation Parameters worktable.

Classification sectors are used by Portfolio Management to display the cash allocated by a Cash Allocation process in different formats, or by different groupings. Validation checks can be performed on these classification sectors to ensure the proposed orders do not breach compliance rules. For example, the cash allocated to underlying entities may meet all individual allocation bounds, but collectively may exceed the total amount of money allocated a single asset sector, custodian, or other grouping.

For further detail about the classification sectors, including an example, refer to Classifications, and associated data in the U496: Classification Allocation Parameters.

Note
  • This worktable must be populated with a Classification code prior to populating the U496: Classification Allocation Parameters.
  • The classifications configured here will not be visible in the Portfolio Management page until the CA_LOAD_SECTORS general process is run. Refer below for further details.

 Sample worktable data

Processes that use this worktable

Portfolio Management

  • U495 data is used during the Display Proposed Transactions stage of the process. (Refer to Portfolio Management for further detail of the whole process.) The data in this worktable is used to create the Classifications categories or sectors, which will contain the summed and re-grouped proposed transactions, to be used for validation checks.

 General processes

When data is loaded into this worktable, the CA_LOAD_SECTORS general process must be run, to move records across to the required (java) database tables. When this process completes successfully, the status of the rows will be 'C' (Completed.)

If data is loaded to this worktable via the U495 Take On process, the CA_LOAD_SECTORS general process will be run automatically.
