Worktables

Data within pControl can be viewed via worktables also known as buffers. Worktables are categorised as:

  • Import - for holding data that has come into pControl from an external source. This data is used as an input to a process.
  • pControl - hold the results of one process to be used as a follow on for another process, or for reporting within pControl.
  • Static - data such as mapping data, reference or lookup data.
  • Export - for holding the output of a process. These worktables are also used to facilitate the export of data outside of pControl.

p_Status Fields

Most worktables in pControl contain a p_status field. The p_status value indicates the status of an individual record. pControl processes use the status value to determine whether or not to select a record for inclusion in a process.

Import Worktables

Import worktables hold data that has come into pControl from an external source. This data is used as an input to a process.

U100: Investment Holdings

U101: External Prices

U104: General Ledger

The U104: General Ledger worktable contains items and values that will be loaded to pControl accounts, for example, from an external general ledger or report. The imported data is enriched to allow the journal creation process to run, which will create journals to update pControl accounts. This provides an audit trail of the imported values and any enrichments prior to creation of account movement journals in pControl.
U107: Price Component Adjustments

U108: Units on Issue

The U108: Units on Issue worktable holds details of the units that have been issued to unit holders of the entity.
U109: Unitised Cash Flow

U113: Pending Settlement Transactions

U175: Security Prices

U178: FX Rates

U180: Security Details

U181: Foreign Income Tax

U200: Asset Transactions

U210: Corporate Actions

U280: Benchmark Value Data

U305: Cash Allocation - Liquidity Adjustments

Holds adjustments that need to be made to the liquidity balance of an entity over a specified time period.
U306: Cash Allocation Liabilities

Holds liability values of an entity for a specific date. Three different types of liability can be loaded into this worktable; close of business (CoB), intra-day, and manual adjustment.
U310: Exposure Reallocation Source

Holds details of the assets and their values which are to be reallocated to other entities during processing. This worktable is used by Portfolio Management. For further information on the reallocation of values, refer to Reallocation of Value.
U350: Classification Hierarchy
P135: Import Register

Is used to represent and manage data originating from the table transfer process, manual entry and direct insert from an external system.

The Validate Data process can process multiple batches of data in parallel and can run multiple Enrich/Validate entities on each worktable.
U485: External Child Assets

Holds the code and description of the external child assets required prior to loading the U490: External Look-through. Prior to loading the U490: External Look-through data, the external child assets must first be created. The U485 worktable holds the code and description of these assets. The CA_LOAD_EXT_CHILD_ASSETS needs to be run so that the U485 details can be used to create the external child assets in pControl.

The external child assets once created, are stored in the allocation parameters and allocation records as a pControl id. With the mandatory fields being code and description, these are stored in a node id table, similar to how the classification sectors are stored with a unique node type.
U490: External Look-through

Stores the investment path details or risk factors for the direct assets of a CAR process. These are recorded as a set of external child assets and their weights within the direct asset. As the child assets do not exist in pControl, the U485: External Child Asset data load and associated process, first needs to be run in order to create these assets. This data is used by CAR for Classification Look-through when the look-through weights are externally supplied.
U495: Classification Sectors

The U495: Classification Sector worktable allows one or more classification structures to be created, which classifies investment data by various criteria. These structures may be the same, or can be different to the investment or interfunding relationship structures. This worktable operates in conjunction with the U496: Classification Allocation Parameters worktable.

Classification sectors are used by Portfolio Management to display the cash allocated by a Cash Allocation process in different formats, or by different groupings. Validation checks can be performed on these classification sectors to ensure the proposed orders do not breach compliance rules. For example, the cash allocated to underlying entities may meet all individual allocation bounds, but collectively may exceed the total amount of money allocated a single asset sector, custodian, or other grouping.

For further detail about the classification sectors, refer to Classifications, and associated data in the U496: Classification Allocation Parameters.

pControl Worktables

pControl worktables are typically read-only and are used to hold the results of processes or functions, and to facilitate the export of information that may be required for follow on processing, reporting or for distribution outside of pControl. For example, the results of Valuation (formerly known as MDV) processing may be used as an input to ALV, NAV or Unit Pricing. These worktables may also be used to hold the results of one section of a process, such as validation checks. In some cases, only those validation checks that failed, (i.e. caused an exception) will be written to these worktables; Other times, depending on the selection of function codes and attributes, all validation checks, whether passed or failed will be written here. Some pControl worktables are also used as a log to record each file received into pControl, and the status of those files; or to record each file generated and distributed out of pControl.

A01: Product Validation

Holds the results of 'Product' validation checks which have breached their tolerance, and therefore an exception has been raised.

Note: Product Validation checks can be identified with a 'POV' prefix code.

A02: Product Validation Detail

Holds the results of all 'Product' validation checks, i.e those that have passed their validation check, (status = 'OK') as well as those that have failed their validation check, (status = 'Exception', or other severity level).

Note: Product Validation checks can be identified with a 'POV' prefix code.
A03: Holding Validation

Holds the summary results for an entity's holdings and validation calculations.
A04: Holding Detail Validation

Holds the detailed results for all validations run on an entity's holdings.
A05: Transaction Validation

Holds the results of the transaction validation functions, i.e. those with 'TXN' prefixes.
A06: Transaction Detail Validation

Holds the detailed results for all validations run on an entity's transactions.
A07: Validation

 
A08: Validation Detail

 
A09: Account Balance Validation

Holds the results of account balance validation functions, i.e. those with 'ACC' prefixes. This worktable can be populated when the Account Movement Attribution report is configured.
A10: Account Balance Detail Validation

Holds the detailed results of the account balance validation functions, i.e. those with 'ACC' prefixes.
A11: Capital Flow Validation

Holds the results of Capital Flow validations, i.e. those with 'PRV' prefixes.
A12: Capital Flow Detail Validation

Holds the results of all Capital Flow validations i.e. those that have passed their validation check, (status = 'OK') as well as those that have failed their validation check, (status = 'Exception', or other severity level).

Note: Capital Flow validation checks can be identified with a 'PRV' prefix code.
A13: Category Validation

Holds the summary results of category validations and calculations.

Note: This worktable is only populated, if the NAV Movement Analysis report is configured.

A14: Category Detail Validation

Holds the detailed results for all validations run on an entity's categories. The worktable is populated when the Share Class Impacts (CPAS) report is configured.

A99: Validation Details

Holds the details of all validation checks performed during a process. This includes both validation checks that passed successfully, as well as those that generated an exception.

A100: Dynamic Benchmark

Holds the security level detail of a Dynamic Benchmark.
A101: Overs and Unders

Holds the security level comparison detail of a portfolio to a benchmark.

Static Worktables

pControl Export worktables are typically read only, and are used to hold the results of pControl processing. These worktables facilitate the export of any data, that may be required for analysis or reporting outside of pControl, some of which are described below.

L081: Editor Options

The worktable defines the available options for selection for each worktable field, where the editor type is Worktable Tree Combo, enabling smart filtering.
L184: Account Category

L200: Tolerances

Holds the tolerance values and severity levels, which are used when a validity check is breached. These values are reported in status messages, as well as against the exception itself.
L220: Instrument Type Mappings

 
L222: Code Mappings

Holds general mappings to match external system codes used in the source data, with those used within pControl.
L235: Benchmark Mappings

Holds a list of benchmarks associated to the asset holding of an entity. These are used by some of the Expected Return Calculation Methodologies to calculate an expected price.
L290: Effective Exposure Reallocation

Lists the assets of the plan whose value is to be transferred from one entity to another. Portfolio Management uses this worktable in conjunction with the U310: Exposure Reallocation Source to reallocate the value of the asset listed in the 'Asset Code' field to the 'Destination pControl Code entity' during the Cash Allocation process.

Note: Reallocation is not a transfer of ownership. The title of the asset remains with the 'Source' entity, as listed in the L290: Effective Exposure Reallocation worktable.

L315: Account Code Mapping

The L315: Account Code Mapping worktable assists in the mapping of account balances from the U104: General Ledger. It specifically maps the account codes and account groups for the Analysis Views. The L315: Account Code Mapping worktable also allows for account groups to be excluded in account code mapping records.

Export Worktables

Control Export worktables are typically read only, and are used to hold the results of pControl processing. These worktables facilitate the export of any data, that may be required for analysis or reporting outside of pControl, some of which are described below.

U141: Unit Prices

Holds the unit prices that were calculated or received for an entity.
U142: Investment Holdings Balances

This worktable holds the investment holding balances that were used for a given processing day and price run type.
P606: Constituent Returns

Holds the expected returns and impacts for each constituent of the ‘Expected NAV Constituents’ calculated during the NAV Oversight process. These details explain the calculation of the expected NAV. This worktable is populated when the [EX300] - Export Constituent Structure function is selected in the 'Exports' phase of a pricing process template.
P607: Constituent Totals

Holds the total expected return and impact for each constituent of the ‘Expected NAV Constituents’ calculated during the NAV Oversight process.
U582: Validation Detail

Validation data calculated during an Expected Pricing process or Actual Pricing process, by the [EX120] function, will be exported to this worktable. This includes both validations that passed and failed.
U583: Price Data

The key control indicators of a price record, for a given entity, price run type, and effective date are exported to this worktable, for example, Approval and Release status, users and price location.
U780: NAV Movement Analysis

Holds the NAV Movement Analysis and Oversight Comparison Analysis report data for a price record.
U781: Movement Analysis

 
U782: Account Movement Analysis

The AMAS category records and the individual account records are exported to the U782: Account Movement Analysis worktable when the [EX092] - Export Account Movement Analysis Data attribute is selected. Records are exported in the order they appear on the Account Movement Attribution view.
U783: Distribution Yield Analysis

The DYAS records are exported to the U783: Distribution Yield Analysis worktable when the process is run on a distribution day and the [EX093] - Export Distribution Yield Attribution Data function is selected on the process template. Records display in the worktable in the order they appear in the DYAS view. Existing records in the table are replaced when processes are re-run.
