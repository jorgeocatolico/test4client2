Classification Configuration

A Portfolio Manager may build and maintain classification structures through the Classification Configuration menu. A classification structure enables a Portfolio Manager to set allocation parameters to sectors within the structure, thus complying with tests applied to the assets. Validation checks can be performed on classification sectors to ensure the proposed orders do not breach compliance rules. For example, the cash allocated to underlying entities may meet all individual allocation bounds, but collectively may exceed the total amount of money allocated to a single asset sector, custodian or other grouping.

 Classification structure versioning

Classification structures are likely to change over time and to accommodate this, a revised structure will hold a new version number and effective date. As dual authorisation is active on classification structures, the structure will be active once it has been authorised. Any existing classification parameters will be brought into the new structure and their effective date updated accordingly. If dual authorisation is active on the classification parameters also, any new parameters will be placed into the 'In Progress' state and will require authorisation.

Classifications with more than one version of the same code, can be viewed using the expand and collapse icon to the left of the classification code. The classification with the most recent effective date and the highest version number will be displayed at the top. Other classifications with the same code will be sorted below in descending order.

Where a sector is deleted in the updated classification structure and that sector was used as a denominator over multiple other classification parameters, all classification parameters that used that sector as the denominator will have the denominator removed.

Note

Classifications are filtered by the master filter effective date criteria and therefore the filter will return all classifications with an effective date between the from and to date in the filter.

Modifying Classification Structures

To add a new classification structure
  1. Select Configuration > Classification Configuration from the top navigation menu.

  2. Select Add.

  3. Enter the classification code, name and effective date. If the classification code and name already exist, a second (or follow on) version of the structure will be created.

  4. Select Save to save changes or Cancel to exit without saving changes.
  5. Once saved, the record will appear in the table and will be 'In Progress'.

 

 To add a new sector

  1. Select Sectors from the drop-down menu at the top of the page, followed by Add.

  2. Enter the sector code and name. If the code already exists, a warning will be displayed and a new code will be required. The sector type selected will become the default sector type when the sector is added to a classification structure. The sector type can be changed in the classification structure, if required. The code and type of sector are required fields.

  3. Select Save to save changes or Cancel to exit without saving changes.
  4. The list of sectors will update to include the new sector.

 

 To add a sector to a classification structure

  1. Search for the required classification structure and select the ellipsis followed by View.

  2. Use the search field to find the sector. Select the required sector and drag and drop into the second panel..

    Note

    Adding a sector without a parent code means the sector will default as a child of the overarching classification structure.

  3. It is possible to re-order within the classification tree structure by drag and drop. The classification structure will update to reflect any changes, including moving any children. To change a sector from a parent to a child, drag and drop the sector to the right slightly.
  4. For each sector, identify the type by selecting from the drop-down menu.

    5. Row type

     

     

    Note

    When "aggregate" is selected, the sector must have at least one child sector. A warning will display and the classification structure will not validate until at least one child sector is added.

    Where necessary, an aggregate can be created without holdings by overriding the warnings. This is described further in Managing Exceptions on Sectors without Holdings.

     
  5. Select Save to save changes pending authorisation, or Cancel to exit without saving.

 

 To copy and create a new version of an existing classification structure

It is possible to copy a classification structure and create a new tree, or create a new version of an existing tree.

  1. In the search field, search for the classification structure or scroll through the list of classifications, as appropriate.

  2. Select the check box of the classification that requires to be copied. Select Copy. To create a new tree, set the code, description and effective date. To create a new version of the existing tree, change the effective date and leave the code and description unchanged.

    Note

    It is possible to create a new version at the same effective date as the existing tree, if modifications are required as at the original date.

  3. Select Save to save changes pending authorisation, or Cancel to exit without saving changes.

  4. The new version of the classification structure will open. If the structure is a new tree (with new code, description and effective date), the structure will display as v1. If the structure is a later version, it will display as v2 (or higher).

    Make the required changes to the structure.

    Where editing v2 or higher, there is an option Compare to Latest to open the comparison view. The comparison view shows the differences between the new tree and the prior version.

    Note

    If this is version 1 of a tree, the comparison view will not be available.

  5. se the Pending Changes panel to review changes that have been made. The panel displays and colour codes all edits that have been made.

  6. Select <Save> to commit the changes to the database.

  7. Select the appropriate transition status, i.e. Ready to Authorise from the Pending Changes panel.
  8. Select Cancel to exit the structure builder screen.

 

 To delete sectors from a classification structure

Sectors can be removed from the classification structure using delete icon, but can only be deleted in the following instances:

  • If it is a first version of a classification structure and it is before the structure has been saved. The delete icon will become inactive once a structure has been saved.
  • If the classification is a new version of an existing classification and has not yet been saved, as shown below.

When a classification structure is rejected by the alternate user, the entire structure will be removed and pControl will revert the classification structure to the previous version.

Note

Deleting sectors may impact classification parameters as any classification parameter that uses the sector as the denominator will have the denominator removed. An example is provided under Classification structure versioning.

Understanding Classification Structure with Dual Authorisation

To authorise a classification structure

Use the drop-down in the Pending Changes panel and select the appropriate state. Once authorised, the classification structure is saved as the new version as at the classification structure effective date.

Alternatively, it is possible to transition the record without viewing details of changes made by selecting the appropriate state from the ellipsis menu, as shown below.

To view classification structure pending changes on the ASM

It is possible to view all structures that have pending changes by navigating to the Attention Status Monitor (ASM), in processing mode.

  1. Select the highest entity in the process tree.
  2. From the Processing menu, select Attention Status Monitor.

  3. Select a dashboard containing the Dual Authorisation status stream.
