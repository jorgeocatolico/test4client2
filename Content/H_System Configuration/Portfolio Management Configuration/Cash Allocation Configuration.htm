Cash Allocation Configuration

Web Mode

Cash Allocation processes are driven through the configuration of the processing template and the setting of attributes associated to an entity, described in Process Entities and Data Loads.

Target groups

Target Groups are required when one or more entities within a nominal group have the 'Exclude this asset from Cash Allocation and Rebalancing' attribute selected, and the end weight of the group of assets needs to match the target weight of the group. When an excluded asset is part of a group, a different rebalancing formula is applied to the entities within the target group.

  • Entities with the same value in the 'Target Group' field are said to belong to the same group.
  • An entity can only belong to one Target Group.
  • Destination FROM and Destination TO entities cannot belong to the same group.
  • An excluded asset could belong to a group. When this occurs, that entity's value is not removed from the total investment balance, however an adjustment is made to the calculation of those entities that belong to a group.

 

 Target groups by plan and by originator

Target groups and excluded assets can be defined on a plan basis or by originator. This enables processing where a number of assets that are held across multiple plans, only belong to a target group in some of those plans. It is possible to define the plan or originator for the asset that should be included in the target group.

Note

Within a multi-originator structure, the plan is the parent of an asset. Within a single originator structure, the plan is the originator.

 Spare Cash Reallocation Method 'Move to Target'

If all assets that can be traded have a difference to target of 0.00% (i.e. when there is spare cash on a rebalance) then the following will occur:

  • If the spare cash value is positive, the cash allocation process allocates it to the first asset with a negative proposed order amount that can cover the full amount of the spare cash.
  • If the spare cash value is negative, the cash allocation process allocates it to the first asset with a positive proposed order amount of the spare cash.
  • If no individual asset can cover the full amount of spare cash, the value is maintained in liquidity and the spare cash is not reallocated.

If assets that can be traded have difference to target amounts <>0.00%, i.e. spare cash on cash allocation, then the following will occur:

  • If the spare cash value is positive, the asset that is the most underweight i.e. furthest from the target, will receive the spare cash.
  • If there is enough cash to bring the asset to target and cash remains, the asset that is the second furthest from the target, will be allocated the spare cash.
  • Any remaining cash at this point will remain in liquidity.
  • If the spare cash value is negative, the above behaviour applies to assets that are overweight.
Note

The spare cash reallocation method described differs from Cash Allocation Towards Target allocation method in which the cash is always allocated proportionally.

 

 Spare Cash from Rounding

If spare cash is 0.05 (absolute value) or less, then it is treated as rounding and is allocated to the largest ALP holding. If the value is greater than 0.05 and cannot be allocated to the largest holding, it will remain in liquidity.

 

 Spare Cash Reallocation 'Move to Target' with Excluded Asset and Target Groups

If an entity invests into assets where some assets are excluded, but not within target groups, the value of the excluded asset is not included in the calculation of difference to target (and is based off the refactored targets and weights).

If an entity invests into assets where some assets are excluded and are in target groups, the difference to target amounts are calculated from refactored target and current weights. Where there are target groups, the value of the excluded assets is included in these calculations and pControl will spread the total assets of the group over the non-excluded assets using the refactored actual weight calculation.

 

 Partial Rebalance to Target - Standard Calculation

The Partial Rebalance to Target methodology checks each allocation parameter to determine whether the starting position into the asset specified by the allocation parameter is outside of the tolerances that have been set.

If any investment is identified as being outside of tolerance, an ‘Ideal Order’ is calculated for all investments as follows:

$\small Ideal\ Order\ =\ \left ( Partial\ rebalance\ weight\ in\ asset \times\ Total\ investment\ value\ of\ the\ fund\ \right ) -\ \\ \small Starting\ \$ \ value\ of\ investment\ in\ the\ asset\ $

$\small Partial\ rebalance\ weight\ in\ asset = Percent\ towards\ target\ \times Target\ weight\ in\ asset\ +\ \\ \small \left ( 1\ -\ Percent\ towards\ target\ \times Starting\ position\ \%\ value\right )$
