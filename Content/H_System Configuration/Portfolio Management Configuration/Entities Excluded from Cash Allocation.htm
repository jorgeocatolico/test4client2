Entities Excluded from Cash Allocation

Refactoring target weight calculations

Refactoring occurs when an excluded asset that has a non-zero weight, has its weight assigned to the other assets within the target group, and its weight set to zero. These refactored target weights are then used in the cash allocation process so that no cash flow is assigned to the excluded assets.

The following terms are used in the example below.

$TW_i\ =\ Target\ Weight\ for\ Asset\ i \\ rTW_i\ =\ Refactored\ Target\ Weight\ for\ Asset\ i \\ eTW_G\ =\ The\ Sum\ of\ the\ Excluded\ Assets\ Target\ Weights\ for\ the\ Group \\ nTW_G\ =\ The\ Sum\ of\ the\ Non-Excluded\ Assets\ Target\ Weights\ for\ the\ Group $

Note

When an asset is refactored, its tolerances and bounds will be adjusted by the same proportions. An example is provided under Refactoring Tolerances and Bounds below.

Example

The calculation to refactor the target weight is:

$\small rTW_i\ =\ TW_i\ \times \left (1\ +\ \normalsize \frac{eTW_G}{nTW_G} \right )$

Asset 4 has a weight of 18.3% to be assigned to the other assets within the target group. Asset 4's target weight will become zero and all remaining assets within the group will have their weights refactored using the above calculation.

Therefore, Asset 1's refactored target weight becomes 47.18%.

Note

The Sum of the Non-Excluded Assets Target Weights for the Group, $nTW_G$, could not be zero. However, it would still be possible to continue with the calculation if there was no target group. Examples of calculations where there is no target group are provided below.

 

 Refactoring tolerances and bounds

When an asset is refactored, its tolerances and bounds can be adjusted by the same proportions using the 'Accept/Reject Refactored Tolerances and Bounds' function on the Cash Allocation process template.

When this function is selected on the process template, the cash allocation process will create a wait event and a decision must be taken to accept or reject the refactored tolerances and bounds.

  • If the proposed tolerances and bounds are accepted, the proposed orders will be generated using the refactored target weight, tolerances and bounds. There will be no future prompts to accept or reject as long as the attribute is selected.
  • If the proposed tolerances and bounds are rejected, however the proposed orders will be generated using the refactored target weight, and the original tolerances and bounds. On subsequent cash allocation processes, the message will reappear.

If the function is not selected, the orders will be generated using the refactored target weight and the original tolerances and bounds. No warning or prompt will be displayed.

Example

MG Plan holds 4 assets, each with a target weight of 25% and tolerances and bounds as shown in the table below.

If Asset 1 is excluded, the refactored target weights of the remaining assets are 33%. Accepting to refactor the tolerances and bounds will result in the following changes:

 Rebalancing calculations

The refactored target weights, described above, are used in the rebalancing formula. The calculation for the rebalanced value of an asset is:

$rVAL_i\ =\ \left ( VAL\ \times \ TW_G\ -\ EXC_G \right ) \times \ \large \frac{rTW_i}{TW_G}$

Where:

$TW_i\ =\ Target\ Weight\ for\ Asset\ i \\ rTW_i\ =\ Refactored\ Target\ Weight\ for\ Asset\ i \\ eTW_G\ =\ The\ Sum\ of\ the\ Excluded\ Assets\ Target\ Weights\ for\ the\ Group \\ nTW_G\ =\ The\ Sum\ of\ the\ Non-Excluded\ Assets\ Target\ Weights\ for\ the\ Group \\ rVAL_i\ =\ The\ Rebalanced\ Value\ of\ Asset\ i \\ VAL\ =\ The\ total\ value\ of\ the\ plan\ i.e.\ the\ sum\ of\ the\ value\ of\ all\ assets \\ TW_G\ =\ The\ Sum\ of\ the\ Assets\ Refactored\ Target\ Weights\ for\ the\ group \\ EXC_G\ =\ The\ Sum\ of\ Value\ of\ the\ Excluded\ Assets\ for\ the\ group $

Where there are no excluded assets, there is no requirement to form a target group.

$rVAL_i\ =\ (VAL\ \times TW_i)$

Non-zero excluded assets weights refactored to group

This accounts for excluded assets with non-zero weights and non-zero asset balances and is calculated by first applying the 're-factored weight calculation' followed by the standard rebalance formula. Where a group exists, the weight is proportioned with the group and if a group does not exist, the weight is proportioned across all assets not in a group.
