Processing Cashflows Individually

pControl provides the ability to process cashflows individually when running CAR via the 'External ID' Orders Criteria allocation method. This method requires an external ID on all U110 cashflows (as detailed in Linking Proposed Orders on Interfund Structures using External ID) and the cash is processed in ascending order of the External IDs.

When External ID is selected, the U110: Cashflows and Orders worktable records for the same entity and same effective date are processed individually within the same CAR run as follows:

  • Load Allocation Parameters.
  • Load Investment Holdings.
  • Load Liquidity Account Balances.
  • Calculate Proposed Orders for first cash flow.
  • Where cash allocation method is "Towards Target", Investment holdings will need to be recalculated or reloaded.
  • The CAR process will loop through and calculate proposed orders for the next cash flow, until complete.
  • A set of proposed orders for each cash flow (External ID) will be created within the single CAR run.
  • There will only be one master record per entity per CAR Run Date.
  • External ID will be associated with every proposed order for the entire interfund structure.
Note

The CAR process will always run for the entire interfund structure in a top down manner, regardless of which entities are receiving cash.
