Control Record - Validity Checks

The Validity Checks phase contains functions that are used to check if it is appropriate to be running a process for an entity on a given day.

The Validity Checks processing phase is linked to the 'Process Validations' status stream. If all of the configured checks pass, the process continues to the next phase. If any of the checks fail, the process stops, and will wait for user intervention.

Note

An exception will not be raised for failed validity checks. The processing status message will reflect 'Unable to Process' and the process will stop.

 

 Control Record - Data Loads

The data loads processing phase loads all of the data that is required to perform the processing that has been configured for the entity as defined by the process template.

Note

The data is loaded into temporary work areas - not displayed to users - for processing by pControl rules.

The data loads processing phase is linked to the 'Load Data' status stream. This status stream will be updated by the process to reflect whether the data loads are being processed, have completed successfully, have resulted in an exception (Data Load Exception) or are waiting on data.

 Control Record - Pre-Calculation Validations

The Pre-Calculation Validation (PV) processing phase contains functions that are used to check that the data loaded in the data load phase is appropriate and reconciled.

The pre-calculation validation processing phase is linked to the 'Pre-Calculation Validations' status stream. This status stream will be updated by the process to reflect whether the pre-calculation validations are being processed (Pre PV running), have completed successfully (Validated), have finished with warnings (Warning) or have resulted in an exception (Pre PV Exception).

 Control Record - Calculation Phase

The 'Calculation' processing phase contains rules that determine the values that are used in the calculation. The calculate phase does not have an associated status stream attached as there are no exceptions raised in this processing phase and no checks are being performed. All data should be available to the process from the prior phases.

 

 Control Record - Post Calculation Validation

The post-calculation validation processing phase contains functions that are used to validate the control records (prices, rates etc.) calculated for the entity.

The 'Post-Calculation' validation processing phase is linked to the 'Post-Calculation Validations' status stream. If all the checks pass, processing continues to the enxt phase. If any of the checks fail, an exception is raised which needs to be overridden by a user before the control record can be approved.

 Control Record - Data Export Phase

The 'Data Export' processing phase contains a number of functions that are used to export data, which has been used or calculated in processing, to the pControl export worktables. The data can then be used for analysis and reporting or exporting to external systems. The data export phase does not have an associated status stream attached as there are no exceptions raised.

 

 Control Record - Approval Phase

The control record 'Approval' function in processing is not a formal processing phase but takes place once all other phases have completed to perform any straight through processing. It is possible for example, to configure an entity to do the following once the standard processing phases have completed and a control record has been updated:

  • 'Automatically Approve' the control record - if a control record has no exceptions raised on it during the pricing process, it can automatically be 'Approved' which will mean that dependent funds can continue their processing if they are waiting on this control record being 'Approved'
  • 'Automatically Release' the control record - if a control record has been automatically 'Approved', it can also be configured to be automatically transitioned to a 'Released' state
  • Once released, the control record can move into a state of 'Pending Acknowledgement'.

This functionality provides a means for straight through processing whereby a control record can be generated and if there are no exceptions raised during the process, it can be 'Approved' and 'Released' automatically. Further, if there is a requirement for an external system to send through an acknowledgement / not acknowledged message on receipt of the control record, then this can also be configured.

The approval processing phase is linked to the 'Control Record - Approval' status stream. This status stream is covered in detail below. This is an important status stream from a user perspective as this status stream involves manual transitions / exceptions.
