User Permissions

Configuring user access is completed in the pControl Web Administration application, also known as 'User Permissions' and 'Web Admin'. Configuration includes both setting up new user ids, and assigning access/enabling function to different areas of the pControl web application.

Split into the four tabs, the Web Admin module uses a wizard approach to configuring functionality scope, user roles, user maintenance and entity sets.

The ability to access and view or edit permissions is subject to an individual's security access settings i.e. the user will be restricted within pControl Web Admin to only see the tabs that they have been granted access to. Access is configured via the User Maintenance tab.

Import and Export Function

Using row selection, it is possible to import and export functionality scope groups, user groups and entity sets.

Exporting files

The following example demonstrates how to select and export two functionality scopes. The same process applies to user groups and entity sets.

  • Select one or more item rows for export.

  • Select Export and select from the options to export a single file, or to multiple files i.e. one per object. The files will be exported in Java Script Object Notation (JSON) to the download directory specified in the user's browser settings.

 

 Importing files

The following example demonstrates how to import a functionality scope into pControl Web Admin module. The same process and configuration screen applies to importing user roles and entity sets.

  • Select Import and the import configuration view will open.

     

  • Start the import process with and choose the relevant file to upload.

  • It is possible to check the items within each file and any changes that will be uploaded. pControl will validate each file selected for upload.

     

  • Select Validate to check the file and Close to end the process.

The file will not load to pControl in the following instances:

  • If the file has been modified outside of pControl, the file cannot be loaded. Each export includes a checksum validation. If the checksum fails during import, the file cannot be loaded. The file will return a fatal response that restricts the file from load, shown below.
  • If the file is not of the expected type, the file will be validated with a fatal response. For example, it is not possible to select Import from the Functionality Scope page and load an Entity Set. 
  • If there are no differences between the current file being loaded and what is stored in pControl, the file will not be loaded. pControl will only allow files to be loaded that have differences and are not fatal errors, i.e. checksum or file type failures.
