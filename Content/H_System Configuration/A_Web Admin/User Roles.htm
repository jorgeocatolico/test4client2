User Roles

'User Roles', or user groups as they are also known, are established for ease of administration. Where several users have similar security requirements, they are grouped together using a ‘User Role’ so that changes can be applied to the group as a whole, rather than to each individual user. A user can undertake several roles, i.e. belong to several groups.

The User Roles Matrix

User Roles are a matrix of Functionality Scope, Entity Sets and Processes.

Entity Security and Processes tabs

Entity Security

Entity security refers to the entity sets a user has access to view within pControl web. For the user role being configured, entity sets can be selected that are considered appropriate. Entity security can be configured via the creation of entity sets (in System Administration). Further information is available under Entity Sets.

Processes

The Processes tab lists all processes and could be selected, according role requirements.

 View or edit a user role
  1. From the alphabetical list of role names under the User Roles tab, find the required user role in the list, or use the search box. Select the edit icon , or select anywhere on the row.
  2. The matrix of entity sets and functional roles will be displayed. The combinations selected can be changed by selecting / deselecting the matrix cells.

 Activate or deactivate a user role
  1. From the list of role names under the User Roles tab, find the required user role and select the toggle (activate)(deactivate).

  2. A confirmation prompt will appear. Select Activate, Deactivate or Cancel as appropriate.

 

 Delete a user role
  1. From the list of role names under the User Roles tab, find the required user role and select the delete icon .

  2. A confirmation prompt will appear. Select Delete or Cancel as appropriate.
