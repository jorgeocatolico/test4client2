User Security

 

Description

Value

Notes

Minimum password length

4

Minimum length for a valid user password. Recommended value is >6. The value cannot be set to zero or blank.

Maximum password length

5

Maximum length for a valid user password. The value cannot be set to zero or blank.

Attempts before account is locked

3

Number of incorrect login attempts allowed before account is locked. The account locked toggle will be selected for the user in the user maintenance dialogue box if number of incorrect login attempts has been reached.

Another user with administration rights is required to 'unlock' the account by clicking on the locked toggle and activating it to green.

Days before password expires

30

The number of days a password is valid for. This excludes the first day the user password was changed or int he case of a new user when the account was first created. After the password has expired the user is forced to change it when they next log in using the expired password. The user cannot access the system without changing the password.

Old passwords to compare

2

Number of prior passwords to be stored to check that the user does not reuse a prior password.

Needs alpha and numerics

YES

Whether password must consist of both alpha and numeric characters. Valid values are YES and NO.

Days after password expiry before account becomes inactive

0

Number of days after password expiry before account becomes inactive.

Minimum number of alpha characters

2

Minimum number of alpha characters that the password must contain.

Minimum number of special characters

0

Minimum number of special characters (e.g. %, #, etc) the password must contain. Special characters include all characters that are not alpha-numeric.

Mandatory upper and lower case mix

YES

Whether the password must contain a mix of upper and lower case letters. Valid values are YES and NO.

Check consecutive identical characters in password

3

The value indicates the number of identical characters NOT allowed for a valid password.

A value of '3' indicates that the password is not allowed to have three or more identical characters. These can be any type - alpha, numeric or special.

Note: a value of '1' is allowed to be entered but this logically holds no meaning and is equivalent to having the functionality disabled or setting the value to zero.

Check user id in password

NO

Indicates whether a valid password is allowed to contain the user id character string. This includes the user id character string in reverse order.

as an example, if the user id was equal to 'pControl' then this character string would not be permitted in the password; neither would lortnoCp (the reverse of pControl).

Check user name in password

YES

Indicates whether a valid password is allowed to contain the user name character string in the password. If the user name string is made of 'words' separated by special characters e.g. spaces, apostrophes, underscore etc then each word is checked in the password. For example, if the user name equals 'Jean-Claude Van Hauser' then each of the strings, 'Jean', 'Claude', 'Van' and 'Hauser' would not be allowed in the password.

Valid values are YES and NO.

Current password encryption algorithm

HmacSHA1

Algorithm used to encrypt the password when created. For valid values used in pControl, please refer to pControl User Guide - Advanced Installation.
