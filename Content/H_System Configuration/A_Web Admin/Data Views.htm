Data Views

Data views allow the view of an entity, web page, or a worktable to be changed so as to hide certain fields, or change the order of fields. They can also be used to specify mandatory fields, or add additional security over specified fields by enforcing 'dual authorisation'.

Data Views are configured in System Administration, for use in processing mode, and pControl web. Each data view must be permissioned to a user group for use within Processing mode, or a 'Functional Role' for use within the web.

Note

A data view is required for most web screens, excluding the main dashboard.

Create a Data View

  1. Select Maintain > Data Views

    This will allow you to select the type of data view to be created or modified. The data view is based on the underlying data model associated with the data that is to be displayed.

    Select the data view type from the drop down list at the top.

    • Template / Template - for maintaining entity templates
    • Worktable / Worktable - for defining the layout of a worktable
    • Exception Events / Exception Events - for exceptions or override events
    • Fund Signoff / Fund Signoff - for NAV price records, and the output of other processes which are displayed on the Fund Signoff view.
    Note

    When creating a new data view, there must be a data model already present.

    For templates and worktables, the 'Qualifier' column will show the template name or worktable name as per the underlying data model.

    To find the relevant Qualifier Name

    Within System Administration, select:

    • Templates, and find the required template name for entities:

    • Maintain > Worktable Definitions to find the Code of the required worktable, (left column)

    Attribute

    Description

    A description of the data view. This text is displayed in Web Admin when assigning a data view to a functional role.

    This is an optional field. Value is not displayed anywhere.

    Make New Fields visible by Default

    If selected, any new attributes added to the underlying entity template, (PR_Contact in this example) will automatically be added to this data view.

    This check box is generally not selected on "Summary" data views.

    Default View

    This will be selected if this view is the nominated default view for this data view type. Where several data views are configured for a data view type, the 'default' one will be used when that view is displayed. The user may then select another data view from a dropdown list on the screen, if available.

     

    A data view is configured as the 'default' on the previous ‘Maintain Data views’ selection screen. Once a data view is made the default, the only way to change it is to make a different data view the default.

    Fields Tab

    The Fields tab shows all fields in the data model for the selected data view type. The fields in the model can only be modified within the data model. However, the fields may be formatted for display in the ‘Details’ section below.

    Identifier

    An identifier for the field name

    Note: for an entity template this is the attribute token name, for a pControl worktable, this is the model identifier. This will not change.

    Name

    The field description as will be displayed on the entity

    Note: label from properties dialogue for an entity template or table header for a worktable.

    Hidden

    If selected, this field will not be visible in the data view.

    Column Width

    The width of the column in pixels in the tabular data view.

    Editable

    If selected, then this field will be editable in the screen.

    Note: The Functional Role must also be given 'edit' access in Web Admin for users to be able to edit values in this field using this data view.

    Mandatory

    If selected, this field must contain a value when creating or editing entities or worktables with this data view.

     

    Notes:

    1. A field which is specified as mandatory through the template configuration cannot have this mandatory requirement removed through a data view (i.e. if a field / attribute has been selected as Mandatory in the data view, pControl will NOT allow ‘Hidden’ function to be selected for this field).

    A field which has not been specified as mandatory in the template can be specified as mandatory in the data view e.g. security can be increased through the data view but not decreased.

    2. A field which has not been specified as editable cannot be specified as Mandatory in the data view.

    Four Eyes

    If selected, this field will have four-eyes authorisation applied in the data view, meaning any values entered by one user and saved, must be reviewed and approved by a different user.

     

    Note: If independent authorisation has been selected as a function on the entity template, the data view will default to four-eyes for all attributes of the entity. In this case four-eyes cannot be removed from any of the attributes of the data view however dual-key can be selected. This selection will automatically deselect four-eyes as dual-key is considered to be a higher level of security than four-eyes.

    Dual-Key

    If selected, this field will have dual-key authorisation applied in the data view, meaning any values entered by one user and saved, must be re-keyed by a different user to be approved.

     

    Note: If independent authorisation has been selected as a function on the entity template, the data view will default to dual key for all attributes of the entity.

    Filterable

    Enables the filtering of values within this field.

     

    Notes: This functionality only applies to the web application.

    When a worktable with filtering configured is first viewed, the data will not be filtered, until the user selects values to filter by.

    Editing Format Mask

    Indicates the mask format for the data value e.g. #0.00 or ##0.0000.

     

    Note: This field is only relevant when the field is marked as ‘editable’.
    Groupable

    This column is only applicable to data views over the 'Exceptions' view in Web mode. If selected, the field can be included in the list of 'groupable fields' so as to group exceptions by the values in this field.

    Style

 Apply access permissions to data views
  1. Select the edit icon next to the 'Role Name' you wish to apply data view permissions to.

  2. On the 'Web Views' tab, ensure 'Data' and 'Library' are selected in the first panel.

  3. Whilst still on the 'Web Views' tab, scroll across to the 'Data' and 'Library' columns at the right hand side.

    • If applying permission to a worktable data view, scroll down through the 'Data' column, and select the check box next to the worktable name.
    • If applying permission to an entity data view, scroll down through the 'Library' column and select the check box next to the entity name.

    Note: This refers to the pControl worktable name, or entity name, not the name of the data view.

  5. From the 'Data Views' tab, select the data model from the drop down menu, e.g. Worktable, Template, ExceptionEvent, FundSignoff etc

    Deselecting a check box will remove that access type.

    For 'read-only' access, only the 'View Enable' checkbox should be selected.

    Note: The names displayed in the 'Data View Code' and 'Data View Description' columns refers to the data view name configured in System Admin.
