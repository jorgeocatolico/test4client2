Data Scope

Data Scope refers to the selection of entities a user has access to view, within the pControl web application.

Data scope is configured via the creation of a 'Collection Group' and static / dynamic collections. Collections groups are configured within System Admin; static and dynamic collections are configured within the processing module. If an entity is included within a static or dynamic collection, the user will be able to view all processes and data related to that entity.

Note

For further detail on configuring collection groups, please refer to System Administration, and Processing Vol I user guides.

The 'Permissions' check box must also be selected on any collections, to enable the entities within that collection to be displayed in the pControl web module.

 

 

The data set configuration can be managed using the processing tree or alternatively, a data set could be defined as a special collection (does this need to be included?). The placement of an entity within the processing tree determines whether the user has access to the entity or not. The maintenance of an entity set allows the user to select an entity and request to include the entity and all entities below; include just the entity; exclude the entity; or exclude the entity and all entities below.

Examples of Data Scopes:

  • Pension Fund
  • Income Funds
  • Defined Benefit Funds
  • Super funds.
