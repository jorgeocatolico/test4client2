Calendars

Calendar data is required to define valid processing days, and ensure pControl processes do not run on non working days, e.g., weekends and public holidays.

Funds may be priced on a daily, weekly, or monthly basis; Distributions are usually processed less frequently, and globally clients may work to a calendar representing that particular country's public holidays. By having multiple calendars available, pControl can determine which dates are valid processing days for a certain entity and process type.

Calendars can be defined in pControl web via the configuration menu, and are then associated to an entity via the calendar attributes as shown below.
