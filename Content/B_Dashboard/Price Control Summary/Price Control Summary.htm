Price Control Summary

Many pControl processes result in the creation of a record known as a ‘Control Record’. Control records include unit price records, security price records, FX rates and imported data records. All control records are displayed in the ‘Price Control Summary’ view. Additional panels on this page, contain further detail about the processing status of that record, exception events, calculation details and several navigation options for obtaining further details related to the record. There are also several filtering options to enable searching for like control records.

Price Control Summary panel

The Price control Summary panel allows users to monitor in detail the progress of a control record. It also shows the status of each control record within the 'Group' currently selected on the library tree, and shows the current high level status of the process in terms of the states of the various processing phases.

The panel shows all of the control records that have been initialised, calculated or loaded into pControl for the current group of entities selected on the library tree and/or primary filter across the data range selected. The panel displays the state of the flows as they transition to completed.

The columns displayed in the first panel will depend on the data view created and filters selected.

The following table highlights some information that is stored in the control record in the Price Control Summary screen.

Field

Description

Price Run Type

The type of record (price or valuation point) this represents, e.g. daily pricing.

Effective From Date

The date that the price is effective from.

Entity Code

The pControl code of the entity.

Effective Date

The date that the price is to be used for i.e. is effective for.

Run ID

The run id (or version id) for the selected control record.

Currency

The currency of the control record - this is always the base currency of the entity.

Control Record Values

The control record values shown will depend on the processing that has been run for that entity i.e. if MDV is run on a benchmark, the control records will show an index value, if MDV is run on a currency, the control record will display an FX rate, and if unit pricing is run on an entity, the control record will display the various price types calculated (NAV, Buy, Sell and GAV).

The calculations that are completed for a particular entity are configured through the selection of functions on the process template.

Approval Status

The current status of the control record in the ‘Control Record – Approval’ status stream.

Process Phases Status

The status of the processing phases associated with processing which have status streams attached (Process Validations, Data Loads, Interfund Impact etc).

The control records displayed can be filtered by 'Price run Type' and 'Status' using the filter on the first panel.
