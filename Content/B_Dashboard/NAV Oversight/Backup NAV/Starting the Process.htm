Starting the Process

A service provider outage must be declared, and then authorised by a different person for security reasons. Once authorised, the Backup NAV process will commence automatically. At the completion of the Backup NAV processing, regular processing resumes. If the service provider outage extends beyond one day, a second outage must be declared and authorised.

In Web Mode

Declaring an 'Outage'

  1. Declare an outage for a service provider by selecting <Outages> on the top right of the NAV Oversight Dashboard.

  2. Select the relevant service provider and select add and where appropriate, select the relevant fund. All outages require an effective date and a reason.

    Use the remove arrow to deselect a service provider or fund.

  3. Select Save to save the outage declaration or Close to close the window without saving.

    The <Outages> button on the dashboard, changes to red and an Outage Declaration bar will display on the Analytics bar.

 Authorising an outage

An outage awaiting authorisation will display on the dashboard, pending authorisation. An alternative user is required to review and transition the outage declaration, as described below.

  1. Select the red <Outages> or the Outage Declaration bar. Both will navigate to the P900 worktable.

  2. Right click on the outage row to transition the record.

  3. Enter a comment and select <Save>.

The authorised outage will alert all relevant users on the NAV Oversight Dashboard, using a red highlight, as shown below. The Exceptions Overview component of the dashboard, is replaced with the Backup NAV widget, indicating Backup NAV processing has commenced.

 

 

 Reviewing and approving Backup NAV processes

Backup NAV processes are required to be manually reviewed and approved. Backup NAV is calculating a NAV using the same combination of ERMs as used by Expected Pricing.

On the Backup NAV component of the dashboard, select the purple Pending Authorisation section of the donut. This will navigate to the Fund Sign Off view.

Once all the prices have been calculated, the dashboard will return to normal and the Active Outages will show as complete.

Viewing an active outage

It is possible to view live outages by navigating to Outages > Active Outages, or via the top navigation menu Data > Worktables > P900: Outage Declaration.

 

 Multi day outage

If the outage extends beyond one day, the process of declaring and approving an outage, and manually reviewing approving price records is repeated. pControl is able to calculate values for the second day using the 'last good NAV date', 2 days previous, and yesterday's Backup NAV processing.

The P606: Constituent Returns and P607: Constituent Totals worktables hold the expected returns and impacts for each constituent of the ‘Expected NAV Constituents’ calculated during the NAV Oversight process and explains the calculation of the expected NAV.

 

In Processing Mode

Declaring an 'Outage'

In the following example, 'EMEA_SERVICE_PROV' is the company calculating and providing NAV prices for the Milestone All-Cap Equity Pooled Fund. If this service provider was unable to supply prices, an outage is declared, and the Backup NAV process will be initiated.

  1. Declare an outage for a service provider by right clicking on the service provider, and navigating to the P900: Outage Declaration worktable.
  2. Select <Add> to insert a new row into the P900: Outage Declaration worktable.
  3. Right click on the new row, and select <Properties> to open the row, and complete the following fields:
    • Service Provider Code
    • Effective Date
    • Outage Reason

      •  

  4. Select OK to save the changes or Cancel to close the window without saving.

  5. Select <Update> to save the outage declaration.

The outage will now appear on the Attention Status Monitor, alerting users that this needs to be authorised.

 

 Authorising an 'Outage'

Authorisation of an outage must be completed by a different user.

  1. Double click on the lowest level item in the 'Attention Status Monitor', i.e. EMEA_SERVICE_PROV.

    This will open the P900: Outage Declaration worktable.

  2. Right click on the outage item, and select 'Outage Declaration (Pending Authorisation) > Outage Authorised.

    The status will then be updated to 'Outage Authorised', and the Backup NAV process for all entities associated with this service provider will be initiated.

Note

Backup NAV processes will either be 'Released' automatically, or they may require manual review and releasing, depending upon the configuration of the entity.

The 'outage declaration' will remain on the Attention Status Monitor until such time as all prices have been released.
