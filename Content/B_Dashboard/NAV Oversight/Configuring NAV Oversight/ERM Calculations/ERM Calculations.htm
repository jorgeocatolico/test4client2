[ERM001] Holdings - Index by Key Benchmark

 

Business Description

The day over day return (Daily ROR) for the key benchmark is used to adjust the prior base market value of the associated pool's holdings to calculate today's market value for each holding. The return of the key benchmark will also be used to represent the holding's total return for the final expected NAV calculation

Component

Holdings

Allowable Category Codes
  • AST - Investment Assets
  • CASH - Cash Asset
  • EQ - Equities
  • FI - Fixed Income
  • FFX - Spot Foreign Exchange and Forward Foreign Exchange
  • FU - Futures
  • OPT - Options
  • PROP - Property
  • STMM - Short Term Money Market
  • TRUSTS - Trusts Listed and Unlisted

Prerequisite Functions
  • [DL080] - Load Benchmark Values
    • This function will load the benchmark return values from the Market Data Validation (MDV) process for the 'Key Benchmark' for the same effective date, and the same price run type, as well as prior pricing point and same price run type.

Methodology Attributes

Entity Attributes

    This attribute may contain many Benchmark entities, however only one can be identified as the 'Key Benchmark'.

 

  • [DL080 - Load Benchmark Values
    • The Benchmark 'Price Run Type' value in the [DL080] attribute, is also used to obtain the validated benchmark record for the key benchmark, on the corresponding effective date, and for this price run type, to be used in the calculation of the Expected Price
    Where the Mandatory flag is selected, and the required benchmark values are not yet available, the Expected or Backup NAV process will wait until the benchmark values are available.
    .

Processing Requirements

The Key Benchmark's Return for the current effective date. (Benchmark Return is also known as the Total Benchmark Movement.)

$\small Key\ Benchmark\ Return\ = \large \left ( \frac{Current\ Value}{Prior\ Value} \right )\normalsize -1$

$\small Where\\ \small Current\ Value\ =\ Benchmark\ Return\ for\ the\ current\ effective\ date\\ \small Prior\ Value\ = Benchmark\ Return\ Index\ for\ the\ previous\ pricing\ point$

 

Calculate the Expected Market Value for each holding in each of the above category codes:

$\small Expected\ Market\ Value = Prior\ Day\ Base\ Market\ Value\ ( 1 + Key\ Benchmark\ Return)$

[ERM002] Holdings - Index by Dynamic Benchmark

 

Business Description

The day over day return (Daily ROR) of the key benchmark, or security price for each holding is used to calculate the weighted return for each holding. All holdings are then summed to give a total return which is used in the Expected NAV calculations.

  • TRUSTS - Trusts Listed and Unlisted

Description

Generated Trade

DIVIDEND

A stock dividend is an event in which a corporation distributes a payment to shareholders in the form of shares of stock, as opposed to money, while increasing the number of shares.

Note: It is assumed that for a stock dividend, the number of shares issued per existing number of shares will be provided. If an amount is provided it is assumed to be a cash dividend

If provided, this represents the number of “Ratio new” shares distributed for each “Ratio old” shares held

e.g. If Currently hold 100,000 shares, and 1 share is being issued for every 32.34 shares held

This will be a FOP trade.

Units for trade =

$\small Existing\ units\ held\ in\ asset\ \times \normalsize \left ( \frac{Ratio\ new}{Ratio\ old}\ \right )$

Example:

$\small Units\ for\ trade\\ \small =\ 100,000\ \times \normalsize \left ( \frac{1}{32.34}\ \right )\\ \small =\ 3092$

Note: units truncated, as only whole shares are issued.

Transaction Key = INCOME

CA Type = DIVIDEND

SPLIT BONUS

A Stock Split takes place when a company decides to increase the amount of its outstanding shares while decreasing the nominal share price proportionally, thus keeping the market capitalisation of the company constant.

A company with 10,000 shares trading at $10 can split into 50,000 shares at $2

This represents the number of shares before the issue (Ratio Old) ÷ the number of “Ratio New” shares after the issue.

e.g. If 10,000 are split into 50,000 shares, e.g. 1: 5, then the number of new shares for an investor that currently holds x shares will be:

$\small =\ n\ \times \normalsize \left ( \frac{Ratio\ new}{Ratio\ old} \right )\small -\ n$

$\small =\ 10,000\ \times \normalsize \left ( \frac{5}{1} \right )\small -\ 10,000\\ \small =\ 40,000$

This will be a FOP trade.

$\small Units\ traded\\ \small =\ A03.units\ \times proration\ \times \normalsize \left ( \frac{Ratio\ new}{Ratio\ old} \right ) -\ \small A03.Units$

Transaction Key = PURCHASE

CA Type = SPLIT_BONUS

REVERSE SPLIT

A “Reverse Stock Split” is the exact opposite of a stock split. The company decides to decrease the amount of its outstanding shares while at the same time increasing the share price proportionally, keeping the market capitalisation unchanged.

This represents the number of shares before the issue ÷ the number of shares after the issue

Calculation is the same as for a Split

e.g. If 20,000 are split into 2,000 shares i.e. 10:1, the number of shares “sold” would be:

$\small =\ 20,000\ x\normalsize \left ( \frac{1}{10} \right )\small -\ 20,000\\ \small =\ 18,000$

This will be a FOP trade.

$\small Units\ traded\\ \small =\ Abs\ \left (A03.units \right )\ x\normalsize \left ( \frac{Ratio\ new}{Ratio\ old} \right )\small -\ A03.Units$

$\small =\ Abs\ \left (20,000\right )\ x\normalsize \left ( \frac{1}{10} \right )\small -\ 20,000\\ \small =\ 18,000\ units$

Transaction Key = SALE

CA Type = REVERSE_BONUS

EXCHANGE OF SEC FOR SEC

There are 2 types of share exchanges:

  • Share Exchange Transaction – Two or more companies merge and the corporate action involves only an exchange of shares among the companies.
  • Symbol Change –One company changes its security ticker symbol. Although there is no change in the cost basis, it is a best practice to move the positions from one symbol to the other to account for the name change as the market will reflect valuations on the new symbol. Ticker Symbol changes also include the change of the asset id code which drives the need for this type of processing.

Calculation is similar to splits and reverse splits without the subtraction, as a purchase of the new asset, and sale of old asset occurs.

$\frac{No\ of\ shares\ before\ issue}{No\ of\ shares\ after\ issue}$

e.g. If 20,000 are exchanged into 2,000 shares of the new asset i.e. 10:1, the number of shares “sold” would be 20,000 and “purchased” would be:

$\small =\ 20,000\ x\ \normalsize \frac{1}{10}\\ \small = 2,000$

This will be 2 FOP trades.

 

Transaction Key = SALE

$\small Units\ Traded\ =\ A03\ or A05.units$

 

Transaction Key = PURCHASE

$\small Units\ Traded\ =\ A03\ or A05.units\ \times proration\ \times \normalsize \left ( \frac{Ratio\ New}{Ratio\ Old} \right )$

$\small =\ 20,000\ \times 0.5\ \times \normalsize \frac{1}{10}\\ \small = 1,000$

Note: The PURCHASE is processed with the new asset code , i.e. U210.Intermediate Asset Code

SPIN OFF BONUS

A spinoff is the creation of an independent company through the sale or distribution of new shares of an existing business or division of a parent company. A spinoff is a type of divestiture. The spun-off companies are expected to be worth more as independent entities than as parts of a larger business.

A corporation creates a spinoff by distributing 100% of its ownership interest in that business unit as a stock dividend to existing shareholders.

The number of “Ratio new” shares of the new company distributed for each “Ratio old” shares held of the existing company

e.g. 100,000 shares of the existing company are held, and 1 share of the new company is being issued for every 32.34 shares held of the existing company.

This is very similar to a stock dividend however the stock purchase transaction is on the new company i.e. the new asset code.

    [ERM016] Holdings - Forward Rate Interpolation

     

    Forward Exchange Contracts, also known as Currency Forwards or FX Forwards are an agreement between two parties to exchange two designated currencies at a specific time in the future. Non-Deliverable Forward or NDFs are cash settled forward contracts meaning the notional amounts of the currencies in the contract are not exchanged on settlement, but rather the profit and loss on the transaction is exchanged in a designated settlement currency. NDFs are commonly used for hedging or for trading in illiquid or restricted currency markets.

    FX Forwards and NDF rates are published for standard periods such as 1 month, 2 months, etc. or 30 days, 60 days, 90 days, etc. However, Forwards, and NDFs can be traded to settle on any date. Therefore, if a price is needed for an “odd date” between the standard periods, the price is interpolated. Interpolation is a way to determine the value of the forward, using spot or forward rates to estimate the future value of that holding.

    The Expected Return Methodology for Holdings - Forward Rate Interpolation reprices each leg of the Forward FX or NDF holding during the Expected Pricing/Backup NAV process, based on current day forward exchange rates, and future dated rates using linear interpolation.

    Note

    Linear Interpolation is also referred to as Straight Line Interpolation.

     

    Business Description

    The calculation will reprice each leg of the Forward FX security based on current day forward exchange rates using linear interpolation.

    The difference in value from prior day will then be divided by the pool's last known known Net NAV to calculate the return. The day over day return of each holding will be summed, and used for the Holding Constituent return in the Expected NAV Calculation.

    1. If Local Currency = Base Currency of the Holding, then price = 1
    2. If Maturity date is null, the price is staled, i.e. price = A03.price_local_cur
    3. If NDF Fixing Date is null, price is staled, proceed to step 5, else move to step 4
    4. If NDF Fixing Date $\small \leq$ Effective Date, the Price = Spot Rate with the effective date = NDF Fixing Date, else proceed to step 5
    5. If Days to Settlement < 3, then price = Spot Rate. The effective date is equal to the last known NAV date, else proceed to step 6
    6. Get the FX Forward rate for the Base:Local currency for the Maturity date.

    Where

    $\small If\ Days\ to\ Settlement\ =\ Days\ in\ Period\ Near,\ then\ Holding\ Price\ =\ Forward\ Rate\ Near$

    $\small If\ Days\ to\ Settlement\ =\ Days\ in\ Period\ Far,\ then\ Holding\ Price\ =\ Forward\ Rate\ Far$

    $\small Holding\ Price\ =\ Forward\ Rate\ Near\ + \left ( Forward\ Rate\ Far\ - Forward\ Rate\ Near\ \right )\ \times \normalsize \left [ \frac{\left ( Days\ to\ Settlement\ - Days\ in\ Period\ Near \right )} {\left ( Days\ in\ Period\ Far\ - Days\ in\ Period\ Near \right )} \right ]$

The increase from 30 days to 39 days will be a 9/31 portion of the increase from 30 days to 61 days.

    1 month rate (30 days) = 8.0

    2 month rate (61 days) = 8.5

    $\small Holding\ Price\ =\ Forward\ Rate\ Near\ + \left ( Forward\ Rate\ Far\ - Forward\ Rate\ Near\ \right )\ \times \normalsize \left [ \frac{\left ( Days\ to\ Settlement\ - Days\ in\ Period\ Near \right )} {\left ( Days\ in\ Period\ Far\ - Days\ in\ Period\ Near \right )} \right ]$

It provides the ability to use the reported income distribution rate from the last known NAV date as the rate for calculating the expected NAV or for calculating the NAV and reporting that rate during an outage as well as to use that rate to report to TA's for income distribution. Where the outage is a multi-day outage, the rate would be used during each day of the outage.

    The rate for this process will be taken from the U101: External Unit Prices worktable, from the last known NAV date and used to calculate the impact to the fund’s NAV price for the distribution’s value impact and will also be the reported rate exported to the U141: Unit Prices for downstream outgoing reports.
