ERM016: Holdings - Forward Rate Interpolation

Forward Exchange Contracts, also known as Currency Forwards or FX Forwards are an agreement between two parties to exchange two designated currencies at a specific time in the future. Non-Deliverable Forward or NDFs are cash settled forward contracts meaning the notional amounts of the currencies in the contract are not exchanged on settlement, but rather the profit and loss on the transaction is exchanged in a designated settlement currency. NDFs are commonly used for hedging or for trading in illiquid or restricted currency markets.

FX Forwards and NDF rates are published for standard periods such as 1 month, 2 months, etc. or 30 days, 60 days, 90 days, etc. However, Forwards, and NDFs can be traded to settle on any date. Therefore, if a price is needed for an “odd date” between the standard periods, the price is interpolated. Interpolation is a way to determine the value of the forward, using spot or forward rates to estimate the future value of that holding.

The Expected Return Methodology for Holdings - Forward Rate Interpolation reprices each leg of the Forward FX or NDF holding during the Expected Pricing/Backup NAV process, based on current day forward exchange rates, and future dated rates using linear interpolation.

Note

Linear Interpolation is also referred to as Straight Line Interpolation.

 

 

 

 

Expected Return Methodology Calculation: ERM016: Holdings - Forward Rate Interpolation
Business Description

The calculation will reprice each leg of the Forward FX security based on current day forward exchange rates using linear interpolation.

The difference in value from prior day will then be divided by the pool's last known good Net NAV to calculate the return. The day over day return of each holding will be summed, and used for the Holding Constituent return in the Expected NAV Calculation.

Interpolation

For each Holding with the category code of FWD:

  1. If Local Currency = Base Currency of the Holding, then price = 1
  2. If Maturity date is null, the price is staled, i.e. price = A03.price_local_cur
  3. If NDF Fixing Date is null, price is staled, proceed to step 5, else move to step 4
  4. If NDF Fixing Date $\small \leq$ Effective Date, the Price = Spot Rate with the effective date = NDF Fixing Date, else proceed to step 5
  5. If Days to Settlement < 3, then price = Spot Rate. The effective date is equal to the Last Known NAV date, else proceed to step 6
  6. Get the FX Forward rate for the Base:Local currency for the Maturity date.

$\small Holding\ Price\ =\ Forward\ Rate\ Near\ + \left ( Forward\ Rate\ Far\ - Forward\ Rate\ Near\ \right )\ \times \normalsize \left [ \frac{\left ( Days\ to\ Settlement\ - Days\ in\ Period\ Near \right )} {\left ( Days\ in\ Period\ Far\ - Days\ in\ Period\ Near \right )} \right ]$

Where

$\small If\ the\ Days\ to\ Settlement\ =\ Days\ in\ Period\ Near,\ then\ Price\ =\ Forward\ Rate\ Near$

$\small If\ the\ Days\ to\ Settlement\ =\ Days\ in\ Period\ Far,\ then\ Price\ =\ Forward\ Rate\ Far$

 

Note: the interpolation calculation uses calendar days with no non-working days, meaning weekend or holiday calendars are not required for the calculation.

Example

39 Day Rate

A 39 day FX Forward falls between the 1 month and 2 month standard periods. . Linear interpolation assumes the interest rates increase or decrease steadily from the 1-month rate to the 2-month rate. The increase from 30 days to 39 days will be a 9/31 portion of the increase from 30 days to 61 days.

1 month rate (30 days) = 8.0

2 month rate (61 days) = 8.5

$\small Holding\ Price\ =\ Forward\ Rate\ Near\ + \left ( Forward\ Rate\ Far\ - Forward\ Rate\ Near\ \right )\ x\ \normalsize \left [ \frac{\left ( Days\ to\ Settlement\ - Days\ in\ Period\ Near \right )} {\left ( Days\ in\ Period\ Far\ - Days\ in\ Period\ Near \right )} \right ]$

Holding Price for 39 day rate = 8.0 + (8.5 - 8.0) x [9/31]

Holding Price for 39-day rate = 8.14516

 

Outputs

A03: Holding Validation

For each holding:

  • A03: Local Cur Price = Calculated Interpolated Rate
  • A03: Value Cur Local = Local Expected Market Value (Prior Actual U100 Units * Calculated Interpolated Rate * Prior Actual U100 Pricing Factor)
  • A03: Clean MV Local Cur = Local Expected Market Value
  • A03: FX Rate Cur = Current Day Internal exchange rate for Local/Base currency pair (set to prior day value if FX Rate not found)
  • A03: Value Cur = Expected Market Value (per calculation in P606 below)
  • A03: Clean MV Cur = Expected Market Value

P606: Constituent Structure

For each holding:

  • pControl Code
  • Effective Date
  • Price run Type
  • run ID
  • Constituent Type = "Holding"
  • Category Code = A03: Category Code
  • Asset Code = A03: Asset Code

  • Expected Value = Units/Market Price (For a forward holding, the units represent the value in local currency to be bought or sold at maturity date)

  • Expected Return = Value Difference / Previous Value
  • MV Impact - Value Difference / Last Good NAV
  • Expected Return Methodology = Description of Expected Return Methodology
  • Comment = Settlement Date = DD-Mmm-YYYY (Maturity Date of the Forward holding)
