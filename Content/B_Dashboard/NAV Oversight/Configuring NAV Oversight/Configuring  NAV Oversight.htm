Configuring NAV Oversight

Before running a NAV Oversight process in pControl, configuration must be completed. This task includes:

  • Determination of the correct process template for Oversight entities
  • Configuration of Oversight Pool, Oversight Class or Pricing entities
  • Population of data requirements of NAV Oversight processing i.e. worktable data
  • The use of the ‘Category Hierarchy’ i.e. the mapping of price components to Categories
  • Configuration of the relevant validation entities
  • Setting tolerances for the validations.
NAV Oversight Process Entities

NAV Oversight can be run on Oversight Pool, Oversight Class or NAV Oversight entities. There are also several other entity types used to support the processes as listed in the table below.

Icon Entity Type Description

Expected Pricing Methodology

This entity houses one or more 'Expected Return Calculation Methodology' entities which are used to calculate the expected NAV for a given set of categories. The sum of all calculated categories gives the expected NAV.
Expected Return Calculation Methodology

An entity in the Processing mode associated to a rules file in the Configuration mode housing the calculation formula. This entity contains a list of category types which the formula can be applied to.
Oversight Class

Oversight Class entities are used to represent the class (fund / portfolio) that an external service provider is providing NAV services for, and therefore oversight pricing processes must be run for to verify the NAV values.
Oversight Pool

Oversight Pool entities are used to represent the pool of assets that an external service provider is providing NAV services for, and therefore oversight pricing processes must be run for to verify the NAV values.
Oversight Service Provider

This entity represents the external company providing NAV services. If this external company has an outage, and pControl has to go into Backup NAV mode, the 'outage declaration' is made via this entity.

The values and settings of an entity’s attributes, in conjunction with selection of functions on the process template, will affect the behaviour and results of a NAV Oversight process.

 

 Configuration of Entities

Configuration of entities refers to the selection of function check boxes on a process template to turn those functions on; and then associating the process template to an entity; populating drop down boxes, or other attributes on the entity to support those process functions. When a process is run, only the selected functions will be performed.

A list of the most common functions used in the NAV Oversight process, are included on the Pricing Process page. For further detail on each of these functions, refer to the Unit Pricing or Asset Level Validation user guides.

 

 Worktable Data

The main worktables referenced by the NAV Oversight process are listed below:

Import Worktables

Static Worktables

Static worktables are used to hold static information such as mapping data.

Export Worktables

Export worktables hold the end results of processed data. Data in these worktables can be exported from pControl, or can be used in follow on processing, for reporting, or distribution.

pControl Worktables

pControl worktables are used to hold the results of processing as required for follow on processing or reporting within pControl, or for distribution outside of pControl. These worktables usually can't be manually updated via the user interface.

 

 Category Hierarchies Validation Entities

The 'Pre-Calc Validation' phase, and 'Post-Calc Validation' phases of the Expected Pricing and Actual Pricing processes contain a number of validation checks that can be performed on data coming into the process, or the results of a process. Each validation check is represented as a single check box on the process template, and is linked to a corresponding entity with the same function code, as shown below.

Information on types of validation entities, the entity attributes and an example are provided under Validation Entities.

Most Validation entities will already be configured in the Processing module, at the time of implementation. Additional validation entities can be configured if required.

 

 Validation Checks for Analysis Views

When validation checks on analysis views are performed, pControl performs a calculation. The output of this calculation is compared to teh tolerance values listed in the L200: Tolerances to determine if the value calculated is acceptable, or the severity of the breach of validation check.

Further information on each of the validation checks for analysis views are included in the Validation Checks section.
