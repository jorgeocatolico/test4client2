Category Hierarchy for AMAS

The Category Hierarchy used by the AMAS report is configured in the U350: Category Hierarchy worktable, and is used to order the categories within the AMAS report, as shown below. The U350: Category Hierarchy worktable also defines the components of a price which will be used to compare one price to another. Prior to use with NAV processes, the category hierarchy must be defined in the worktable to allow the movement in net asset value and assets and liabilities to be categorised according to internal classifications. A number of standard categories have been defined in the U350: Classification Hierarchy worktable for use in the AMAS report and can be recognised with the '_AMAS' suffix.

 

The following table shows the standard AMAS category hierarchy. These values can be amended, or added to, depending on a user's access, either by manually updating, or uploading a file to the U350: Category Hierarchy worktable.
