Data Mapping for Analysis Views

Data is mapped for each of the analysis views via the U104: General Ledger, L315: Account Code Mapping, L184: Account Category and U350: Category Hierarchy worktables. An example of data mapping for AMAS is shown below, however, the same approach is used for all analysis views.

The pControl Account is mapped from the U104: General Ledger to the L315: Account Code Mapping. The L315: Account Code Mapping allows for accounts to be mapped to account groups, where relevant and provides an option to exclude in account codes. The account group from the L315: Account Code Mapping is used in the L184: Account Category, where the category code is identified. The category code is subsequently used in the analysis view, and the hierarchy of category codes can be defined in the U350: Category Hierarchy.
