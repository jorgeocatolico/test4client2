The Oversight Comparison Analysis (OMAS) report is generated from the Daily Pricing process. pControl will then use the pricing data from the daily and expected pricing processes to populate the corresponding categories within the OMAS report.

The OMAS takes the results from the NMA for the daily and expected prices and, for each price type, shows the movements in the unit price components as well as the movement in the price and a comparison against the entity’s key benchmark. A comparison is then performed between the actual and expected prices, for each category that makes up that price.

The OMAS view is accessible from the All Prices view in pControl. It is only available for Daily-priced entities. An example is shown in the figure below:

 

The Daily Pricing process will populate the OMAS if the [VA700] Generate Oversight Movement Attribution attribute is selected on the Daily Pricing process template.

Oversight Comparison Analysis (OMAS) Specification

The Oversight Comparison Analysis view is divided into a hierarchy of categories, similar to the NAV Movement Analysis report. The data and calculations that are used to populate each of the categories or category types on the report are detailed in the table in this link OMAS Specification Details.

Column definitions are also provided.

  • Tolerance Value (bp) - this is the value that was breached by the validation performed on this category (if applicable)
  • Tolerance Level – this is the level of breach from the validation performed on this category (if applicable)
  • Summary Tolerance Level – this is the maximum breach that occurred for all validations
  • Summary Exception Count - this is the total number of validation breaches (Warnings and above) that were raised by the process.
