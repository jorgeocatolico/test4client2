Validation Checks

In addition to the analysis views displaying the breakdown, or movement in a NAV value, or movement between classes, they are also created so that further validation checks can be performed on the values within the analysis reports, to determine if the values calculated are within, or outside of tolerances.

When validation checks are performed, pControl performs a calculation. The output of this calculation is compared to L200: Tolerance values to determine if the result is acceptable, (within tolerance), or an exception is to be raised, i.e. tolerance has been breached.

Validation Check Example

Example

The example shows, the [CAT012] validation check has raised an L3 Exception against the FUNDUP_AMAS_01_POOL entity as the impact value, is greater than the tolerance value set in the L200: Tolerance worktable.

The impact value is calculated by:

$\small ABS \left(Unit\ Price\ Movement\ (UPR)\right )\ -\ Unit\ Price\ Movement\ breakdown\ (UPM)$

and the tolerance value is set as 25.00bp for an L3 Exception.

The description section of the analysis views page provides detail on the tolerance value at entity level as well as the unit price movement value (UPR_AMAS) and the balance sheet movement value (UPM_AMAS).

The L200: Tolerance worktable shows the tolerance values that trigger each severity status.
