Share Class Impacts (CPAS)

However, all class movements should be able to be attributed to factors such as different fee levels, distributions, capital flow dilution, rounding of prices, or different local currency class or hedging and if all of these impacts are stripped out to generate a ‘clean return’, that clean return across the classes should be the same. The Share Class Impacts view details the impacts that affect each class’ return along with each class’ clean return so that comparisons can be made against a nominated master class.

The Share Class Impacts view relies on rules segments for defining the comparison criteria used for individual price components. These rules segments can be found in the [UPC195] - Calculate Price Impacts function on the process template. Any differences calculated are then compared to tolerances defined in the L200: Tolerances worktable.

The Share Class Impacts view

The Share Class Impacts view is accessible via the navigation described in Using the Analysis Views from the Exceptions screen, or via the Fund Overview ellipsis on the Fund Sign Off screen.

Worktables

The Share Class Impacts report utilises NAV prices as well as values and mappings from the following worktables. Examples of data mappings are provided in Data Mapping for Analysis Views.

U101: External Unit Prices

The U101: External Unit Prices worktable provides the NAV price for the Share Class Impacts calculations.

 

 U104: General Ledger

The U104: General Ledger worktable contains items and values that will be loaded to pControl accounts.

 

 L315: Account Code Mapping

 

 L184: Account Category

Further information is provided in L184: Account Category.

The L184: Account Category maps the account group to the Share Class Impacts category. Only categories that have "Clean Price Analysis" identified in the L184 category field, will be included in the Share Class Impacts (CPAS) view. Setting the category field associates the account groups and linked category codes to CPAS.

 

 U350: Category Hierarchy

The U350: Classification Hierarchy worktable requires to be configured with asset categories and their hierarchies.

 

 A14: Category Detail Validation

One record for each price type calculated by the [UPC195] - Calculate Price Impacts and validated by the is output to the A14: Category Detail Validation worktable.

 

Functions

The following functions determine whether the Share Class Impacts report is produced and exported the analysis view; and define which rules segments to be executed when creating the CPAS

  • [UPC540] - Calculate Clean Price ROR
  • [UPC195] - Calculate Price Impacts
  • [VA590] - Generate Clean Price Analysis Data

The following are validation checks that can be performed to check the impacts does not exceed a specified tolerance.

  • [CAT008] - Impact Difference
  • [CAT009] - Price Impacts Check
