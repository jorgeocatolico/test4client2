NAV Movement Analysis

The NAV Movement Analysis view

The NAV Movement Analysis view is accessible via the navigation described in Analysis Views from the Exceptions screen, or via the Fund Overview ellipsis on the Fund Sign Off screen.

Worktables

A01: Product Validation

Presents a summary of the validation and calculation results for the fund, such as the calculated GAV, and movement from the previous pricing point. It also contains a summary level statuses for the validations performed.

 A02: Product Detail Validation

Holds the results of all validation checks, i.e those that have passed their validation check, (status = 'OK') as well as those that have failed their validation check, (status = 'Exception', or other severity level).

 A13: Category Validation

Holds the summary results of category validations and calculations.

 A14: Category Detail Validation

Holds the detailed results for all validations run on an entity's categories.

 U350: Category Hierarchy

Functions

The NMA can be generated as part of an Actual pricing or Oversight pricing process, by selecting the following function on the process template:

  • [VA500] - Generate Pricing Movement Analysis Data
    • This function takes the pricing data either from the U101: External Prices or U141: Unit Prices worktable for inclusion in the analysis view.
  • [VA150] - Generate Asset Level Movement Analysis Data
    • This function will include the valuation data from the U100: Investment Holdings worktable, used in the NAV process, for inclusion in the analysis view. This data is used to explain the movement, and impact of investment asset categories, (Equities, Fixed Interest etc), cash positions and transactions, (if applicable) on the price.

Where Expected Pricing is run, the following function can be selected, and the 'Expected Pricing Movement' analysis view will be generated, and displayed within the 'NAV Movement' tab:

  • [VA800] - Generate Expected Pricing Movement Analysis Data

Calculations

NAV Movement Analysis Report Specification

The NMA report specification details the data, calculations and categories used, as well as column definitions.

  • FX Contribution (bp) – this is the amount of the impact that can be attributed to the movement in FX Rates for the values present in the specific category (the FX contribution is included in the Total contribution so this is the amount of the total contribution due to FX Rate changes)

The specific calculation used for the FX Impact is as follows:

$\small FX\ Contribution = \large \frac{\left ( Value\ Previous\ (local))\ \times\ Current\ FX\ Rate \right )\ -\ \left ( Value\ Previous\ (local) )\ \times\ Previous\ FX\ Rate \right )}{Previous\ NAV}$

  • Category Tolerance Level – this is the level of breach from the validation performed on this category (if applicable)
  • Category Tolerance Value – this is the value that was breached by the validation performed on this category (if applicable)
  • Summary Tolerance Level – this is the maximum breach that occurred for all Holdings and Transactions in the specific category (only valid for categories under AST)
  • Summary Exception Count – this is the total number of validation breaches (Warnings and above) that were raised by the process for all Holdings and Transactions in the specific category (only valid for categories under AST).
Note

Not all categories are populated for NAV Oversight where non-security- level data is used to value the portfolio or entity.

The NAV Movement Analysis View is divided into a hierarchy of categories, similar to the other analysis views. The data and calculations that are used to populate each of the categories or category types on the report are detailed in the table in this link NMA Analysis Report Specification.
