Account Movement Attribution

The Account Movement Attribution Analysis (AMAS) screen displays the movement from previous pricing point in the components of the NAV, based on the General Ledger (GL) account balances. The report is presented in both Balance Sheet and Income Statement format. The AMAS consists of categories representing accounts from the U104: General Ledger, for example, assets and liabilities, and income and expenses; as well as additional supporting categories for benchmark values, unit prices, and NAV values. The AMAS provides insights into the day on day movement of these values and the impact of the movement.

As with other analysis reports, Validation Checks can be performed on the output of the AMAS report to compare the impacts of components against tolerance values.

AMAS Report/View

The AMAS is accessible following the navigation described in Using the Analysis Views from the Exceptions screen, or via the Fund Overview ellipsis on the Fund Sign Off screen.

 

Note

The AMAS view shown above and throughout this section is using a pool entity i.e. investment holdings or expenses level. It is possible to apply the AMAS view at class level and it is typically used for [CAT006] Class Price vs Pool Price to check the movement of share class prices is in line with the movement of the pool price.

Worktables

The AMAS utilises values and mappings from the following worktables. Examples of data mappings are provided in Data Mapping for Analysis Views.

U104: General Ledger

The AMAS is populated at Category Code level with account balances from the U104: General Ledger.

 

 U280: Benchmarks

For Benchmark tracking funds, the AMAS displays benchmark values. As with other categories, the current, previous and movement values are displayed, allowing visual comparison of benchmark movement versus NAV price movement. The validation to check that these values are in line is completed in the [POV010] - Price Movement vs Benchmark check. There are two Benchmark categories on AMAS, the IND1_AMAS for Key Benchmark values and the IND2_AMAS for Dynamic Composite Benchmark values.

Benchmark Values are loaded from the U280: Benchmark worktable, validated and price record written during the Market Data Validation process.

Key Benchmarks

For the Benchmark set on the fund entity as the Key Benchmark, current and previous price values will be loaded to the NAV Oversight process and written to the IND1_AMAS category.

Dynamic Benchmarks

Dynamic composite benchmarks can be composed of the underlying assets of an entity and their associated weights, or the benchmarks of those assets. They are used to compare the movement of the entities value against that of the dynamic benchmark. The constituents of the dynamic composite benchmark are mapped in the L235: Benchmark Mapping Worktable. Current and previous price values will be loaded to the NAV Oversight process and written to the IND2_AMAS category.

 

 L184: Account Category

The L184: Account Category maps the account group to the AMAS category. Only categories that have "Account Movement Attribution" identified in the category field in the L184: Account Category will be included in the AMAS view. Setting the category field associates the account groups and linked category codes to AMAS.

It is possible to map a single account to a category without mapping it to an Account Group in which case the Account Group field should be left blank.

 

 L315: Account Code Mapping

 

 U350: Category Hierarchy

The Category Hierarchy used by the AMAS report is configured in the U350: Category Hierarchy worktable, and is used to order the categories within the AMAS report.

 

 U782: Account Movement Analysis

 

 A09: Account Balance Validation

The A09: Account Balance Validation worktable is populated during the pricing process with the category code for the account record. Each category in the AMAS is a collection of accounts; the A09: Account Balance Validation provides a breakdown of those accounts and their balances, movements and impact of those movements.

The A09: Account Balance Validation worktable provides valuable information that can be used to:

  • check records have been mapped
  • to analyse the makeup of the unexplained values on the AMAS
  • and provide data from the impact field for the [ACC004] Account Impact Check which validates the movement against account level.

Where the category code is left blank, the account record has not been mapped and it is these records that makeup the Unexplained Value total.

It is possible to navigate to the A09: Account Balance Validation worktable via the ellipsis on the AMAS view providing the ability to investigate movements within an account category itself. The data in the A09: Account Balance Validation worktable will sort by category accordingly.

 

Functions

The following functions determine whether the AMAS view is populated and the data exported to the U782: Account Movement Attribution worktable:

  • [VA850] - Generate Account Movement Analysis Data
  • [EX092] - Export Account Movement Analysis Data

The following are validation checks that can be performed on the values within the AMAS, to determine if the values calculated are within or outside of tolerances:

  • [CAT006] - Class Price vs Pool Price
  • [CAT011] - Category Level Account Impact Check and [CAT011-PUI] Category Level Account Impact Check (Per Unit Impact)
  • [CAT012] - UPR vs UPM Breakdown
  • [CAT013] - Check for Unexplained Values
  • [CAT014] - Unexplained NAV Impact
  • [CAT016] - Balance Sheet NAV v Capital and Income NAV
  • [ACC004] - Account Impact Check
