Category Hierarchy for Share Class Impacts

The Category Hierarchy used by the Share Class Impacts (or Clean Price Analysis) report, as shown below, is configured in the U350: Category Hierarchy worktable, and is used to classify and order assets within the report, as shown below. The U350: Category Hierarchy worktable also defines the components of a price which will be used to compare one price to another. Prior to use with NAV processes, the category hierarchy must be defined in the worktable to allow the movement in share classes to be monitored in line with each other. A number of standard categories have been defined in the U350: Classification Hierarchy worktable for use in the Share Class Impacts report and can be recognised with the '_CP' suffix.

The following table shows the standard Share Class Impacts category hierarchy. These values can be amended, or added to, depending on a user's access, either by manually updating, or uploading a file to the U350: Category Hierarchy worktable.
