Process Entities and Data Loads

Processing Mode

Portfolio Management relies upon multiple entities, attributes and worktables within the processing mode to run Cash Allocation and Rebalancing.

Cash Allocation Process Entities

The table below describes the entities most commonly used in Cash Allocation and Rebalancing process and Portfolio Management.

Icon

Entity Type

Description

External Product

Represents an externally managed fund.

Group Trust

For accounting / custodial reasons, some assets of a plan are held in a Group Trusts.

Plan

A plan entity defines the highest level of a collection of Portfolios.

Any entity can behave as a plan, so long as that entity type is configured to run the Cash Allocation and Rebalancing process.

Product

A product entity represents a standard unitised vehicle or fund in pControl on which processing may be run (e.g. Cash Allocation processing).

Cash Allocation Process Template

Process templates are entity types that contain all the available functions of a process. Functions are turned on by selecting the check box, and turned off by deselecting. The selected functions will be performed during Cash Allocation processing.

Liquidity Accounts

An entity which defines the internal account/s balances to be retrieved and summed to generate a liquidity balance for an entity.

Liquidity Entity

An entity to represent the amount of cash / liquidity an entity is targeted to hold, i.e. cash at hand.

Each entity running a Cash Allocation process will have an associated liquidity entity.

Notional Portfolio / Model Portfolio

An entity used to represent a set of target investments into assets which are shared across many funds.

Data Loads

The main worktables referenced by Portfolio Management are listed below.

Import Worktables

Where further information is available, a link is provided in the table below.

Import Worktable

Description

U100: Investment Holdings

Holds the investment holding balances (i.e. interfund holdings and direct assets) for an entity for a specified valuation (effective) date.

U108: Units on Issue

Holds the units on issue of an entity.

U109: Unitised Cash Flow

Holds the unitised capital flows into an entity.
U305: Cash Allocation - Liquidity Adjustments Holds adjustments that need to be made to the liquidity balance of an entity over a specified time period.
U306: Cash Allocation Liabilities Holds liability values of an entity for a specific date. Three different types of liability can be loaded into this worktable; close of business (CoB), intra-day, and manual adjustment.

U485: External Child Assets

Holds the code and description of the external child assets required prior to loading the U490: External Look-through.

U490: External Look-through

Stores the investment path details or risk factors for the direct assets of a CAR process.
U495: Classification Sectors Holds Classification sectors which are used to classify investment data by different criteria, and then perform compliance checks on those sectors.
U496: Classification Allocation Parameters Classification Allocation Parameters are used in conjunction with U495: Classification sectors. They define the compliance checks to be performed on proposed transactions.
U498: Investment Parameter Update Holds the allocation criteria, (target weights, tolerances and bounds) used by the Cash Allocation process to calculate how much cash to allocate to underlying assets

U310: Exposure Reallocation Source

Holds details of the assets and their values which are to be reallocated to other entities during processing.
