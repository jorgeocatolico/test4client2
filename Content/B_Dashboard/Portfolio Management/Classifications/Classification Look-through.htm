Classification Look-through

Understanding Classification Look-through

Classification look-through provides the ability to look through a plan's investment structure allowing compliance checking of the underlying child assets on the Portfolio Management classification structure panel.

The classification structure can be populated by:

  • Using externally sourced look-through weights. This is known as External Look-through with data loaded into the U490: External Look-through worktable.
  • Using Investment Allocation Parameters (ALPs) or Investment Allocation Records (ALRs) as the source of weights. This is known as Internal Single Level.
  • Using the Internal Look-through worktable as the source of weights which is considered Internal Multi Level. This uses the P490: Look-through worktable.

The weights and values of the plan's direct assets, (i.e. the investment ALR), are pro-rated by their child asset weights to create a look-through ALR for each child asset. The child assets can be assigned to a look-through classification structure. CAR will then create a classification ALR allowing the child assets to be displayed in a classification and used in compliance checks.

Child assets can be externally sourced holdings or factors, or internal investment structure entities.

Source of Look-through Weights

Data source options

There are four potential sources of the look-through weights:

  1. P490: Internal Look-through worktable.
  2. U490: External Look-through worktable.
  3. Investment ALP: static data.
  4. Investment ALR: CAR process output.

An attribute setting is available on the Cash Allocation tab to determine which type of look-through is to be applied.

Data source options include Allocation records, External, Internal, Strategic allocation parameters or Target allocation parameters.

The chain length level refers to the number of levels to look through down from the asset for the internal and external look-through.

Note

When the Allocation Records, Target Weights, or Strategic Weights option is chosen as the data source, then the chain length level / look through level can only be one.

If a deeper look-through level is required, then the look-through process should be run at a deeper level with the internal data source.

 External Look-through

The external look-through data is loaded into the U490: External Look-through worktable. Similar to the P490: Internal Look-through worktable, the process for retrieving data from the U490 is the same as for the P490 except instead of calculating the look-through data from the investment path, the U490 may hold investment path and other types of look-through weights such as risk factors or asset class components. Therefore, there is an additional field in U490 for a classification tag.

Further information is provided under U490: External Look-through.

 

 ALR Look-through

The ALR look-through (T215) refers to using the output of CAR processing from lower level plans, as the weights in the look-through calculation. Although an Investment ALR could be used to go down multiple levels in the investment structure, the ALR look-through can only be used to view one level down. The internal look-through should be used if multiple levels of the investment structure are to be looked through.

 Aggregating across like child asset

In the case where the Look-through ALR has the same asset and originator ID, then these are aggregated as there is only one place in the tree to map these child assets.

When multiple instances of the same child asset exist, then:

  1. The value and weights fields are summed.
  2. The new value fields are to take new values.
  3. The following same value fields are not to be populated: external_reference, order_id, and transaction_type_id.

Default Asset Classification

Understanding Default Asset Classification

To assist in the maintenance of the Classification Parameters, the default asset classification and a classification check are attributes in the Cash Allocation process template. As the non-leaf nodes of a classification are mainly static and shared across multiple plans, and the leaf nodes for a plan (i.e. the assets) can change on a more regular basis, maintenance of these assets on a per plan basis can be reduced by using the Default Asset Classification functionality.

Sectors that share the same code as the asset are referred to as Default Sectors. Default Asset Classification will automatically assign the asset to their default sector.

There are two attributes in the Cash Allocation Process template to determine asset classification.

Classify Assets

When active, this attribute will ensure that the asset classification process is run as part of the CAR or Start of Day processes.

Check for Unclassified Assets

The default sector refers to a sector with the same code as the unclassified asset. pControl will check to see if a matched sector exists and is type H - holding. If there is no match found, then processing stops for the asset in this classification and a message indicates Asset XXX has no default sector in classification YYY.
