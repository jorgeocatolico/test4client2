Starting the Process

Portfolio Management processes require Cash Allocation to be run via the Processing mode in order to be viewed in web mode. The Cash Allocation process starts with cashflows provided from Registries, Transfer Agencies, Wholesale Clients or any external source. pControl then processes these cashflows through the investment structure such that orders into underlying Portfolios, Assets or External Products are created and can be sent to market.

The Cash Allocation process is designed to run across complex, multi-tiered investment structures. It aims to use straight-through processing to take the cashflow data and invest it into the investment structure to ensure funds are 'fully invested'.

To manage the processing of cashflows through the investment structure, pControl manages the dependencies that exist between funds through its inherent understanding of both the actual investment structure (reflected by investment holdings and registry balances) and also the target investment structure (Classification Parameters).

Note

For further details on the Cash Allocation process, please refer to the Cash Allocation user guide.
