Proposed Transactions

The third panel of the Portfolio Management workbench displays the proposed transactions into the underlying portfolios. pControl produces proposed transaction records based on the Investment Parameters, which are populated in Configuration > Investment Parameters. Once populated, the Portfolio Manager can review the proposed order values, and approve, reject or modify their value.

The new highlight, filter and hide zero values functionality, specific to Portfolio Management is described below.

 Proposed Transactions example
Example

A Portfolio Manager may manage the following portfolios:

  • Australian Shares portfolio
  • International Shares portfolio
  • Australian Property portfolio
  • Australian Fixed Income portfolio
  • International Bond portfolio

Every day, the Portfolio Manager may wish to check each of the portfolios and if they are out of tolerance, consider adjusting the portfolio back to tolerance. The Portfolio Manager may take into account factors like the client's investment guidelines, trading terms of each investment and business partnership.

Using the Filter Classifications function

The filtering of proposed orders enables the Portfolio Manager to easily identify proposed orders for assets within a specific asset class or sub class without having to scroll through scores of orders. This enables the Portfolio Manager to model and make changes of the asset classes and sub-classes within a timely manner.

Using the Highlight Changes function

Applying the highlight changes button, enables the Portfolio Manager or the manager with overall responsibility, for example, a CIO, to identify all trades that have been modified and can view them in a timely and efficient manner. The CIO will be able to identify via the yellow highlighted rows any changes that have been committed and/or validated.

Using the Hide Zero Values function

A Portfolio Manager or a CIO may wish to hide investments that hold a zero value. All investments with a zero total value, in base currency, will be removed from both the investment panel and the classification panel, when this function is selected.

Note: All of these functions can be set to default in System Administration.

 

Changing the Panel View

Using the Filter Classifications function

The Filter Classifications option enables the Portfolio Manager to isolate and therefore easily locate investment records that are directly impacting the classification structure.

  1. Select a plan that has proposed orders that are unapproved.
  2. From the classifications panel, select the check box of the row required. Any children will automatically be selected.
  3. Select the filter classification icon which will highlight blue once selected.
  4. The proposed transactions will reorder, and all associated investment records will be moved to the top of the panel as shown below.
  5. Any records not relating to the selected classification rows will display in grey font.

  6. To return, de-select the filter classifications button

 

 

 Using the Highlight Changes functionality

The Highlight Changes option allows easy visibility of modified data in the current screen by highlighting the modifications in yellow. Modifications include changes to a proposed order where units, amount or weight are modified and then validated. If the resulting order is different to the original proposed order calculated by pControl, the order will be highlighted as a change.

Once selected, the highlight changes icon will display in blue as shown in the image below. The Highlight Changes function can be deactivated by de-selecting the button.

 

 

Modifying Proposed Transactions

To modify proposed transactions

The editable fields on a proposed order will vary, depending on the data view selected. The most common editable fields are:

  • Cash Amount
  • Cash Amount - Asset Currency
  • Order Units
  • Order Weight %
  • Confirmation date
  • Proceeds date
  • User notes
  • Reason code.

Editable fields are indicated with a grey arrow in the top right corner of the field as shown below:

Note

If the above fields are not available in the Proposed Transactions panel, they may be reordered or excluded from the data view. Data views are configured in System Administration.

Proposed Transactions for the 'Liquidity Entity' are not editable. pControl will automatically approve these transactions during processing.

  1. Select the proposed transaction to be modified.
  2. Edit fields as required.

  3. Select to re-run the validation checks over these updated values.

    Note

    The validation checks that will be performed are the same ones that are run during the 'Post Processing Validation' phase of processing.

     

    The Classification panel will indicate any modified values which are outside of their tolerances or bounds in the 'Compliance Check' columns:

  4. Review values outside of tolerance / bounds and repeat steps 1-3, until the proposed transactions are within the tolerances and bounds.
  5. Select to save the proposed transactions.

 

 To clear modified proposed transactions

Manual changes made to Proposed Transactions can be cleared by reselecting the search icon , changing the data view, or selecting a different record in the Investment Summary panel. This will cause the web page to clear any uncommitted value changes, and redisplay the original proposed transactions as calculated by the processing.

Note

If the manual changes have been committed, they cannot be cleared this way. Instead, committed changes must be reversed / cancelled in pControl, or a new transaction entered in pControl for the opposing value.

 

 To save changes

Use the commit icon to save changes made to the proposed orders. Any hard or soft warnings will open a prompt to confirm that the changes should be saved. If proceeding, the record will commit to the database.

Note

Saving records to the database does not transition the records into orders. Proposed transactions can be transitioned following the steps described below.

 

 To approve proposed transactions

If the Portfolio Manager is satisfied with the values on the proposed transactions, the transactions can be transitioned to 'approved' by:

  1. Select the transaction.

  2. Right click and select appropriate transition.

     

Note

It is possible to transition multiple transactions at the same time by selecting the check boxes to the left of the transaction and right clicking.

 

Awaiting Completion state

CAR can run to a completed state while having an additional transition to a final approval state so that proposed orders can be approved once cash has settled for the trades, as opposed to being approved on execution date. The additional end state, 'Awaiting Completion' can be reached via 'Pending Completion'. The 'Include Awaiting Completion Balances' attribute on the CAR processing template is required to be set to true in order to include the balances of proposed orders in the 'Awaiting Completion' state in the subsequent CAR run.
