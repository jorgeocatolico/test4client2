Balance Differences

Where a difference is identified between pControl’s value and an actual bank account value (as supplied from an external source), a balance difference transaction needs to be executed in pControl’s processing module. Once investigated and an appropriate transaction created, the balance difference can be cancelled.

 Creating ‘Balance Difference’ transactions

The actual bank balance value of $5,000 for 25-Jan-2017 is received into pControl via a csv file which is imported to the U365: Cash Balances worktable. The importing and processing of bank account values is completed using the pControl take-on process.

Note

Positive values in the U365: Cash Balances worktable indicate positive bank account values. Negative values indicate negative bank balances.

As soon as the take on and validation processes are completed, and the U365: Cash Balances worktable contains a ‘completed’ record, the ‘Balance Difference’ process will automatically commence. pControl will calculate the difference between the cash holdings, (-$2001) and the U365 value ($5000), and will generate an Income / Expense transaction for $7001, using the transaction type configured on the account.

Note

In the ‘Maintain Income / Expense’ view, a positive value is a payment, and a negative value is a receipt.

 

 Amending 'Balance Difference' transactions
  1. Right click on the Income / Expense transaction and select ‘Properties’.
  2. Click on the padlock icon, it will change to an open padlock. The properties view of the Income / Expense transaction is displayed, and values in black text, such as the ‘Cash Amount’ can be amended if required.
    3. Note

    The padlock icon will only appear from the 'Maintain Income / Expense' view. It is not possible to amend any values from the 'View Income / Expense' screen.

     

  3. Change the required values.
  4. Select <Ready to Auth> when all changes are completed, and to send the change to be checked by a different user, or
    • Select <In Progress> to save the changes and continue working on them later or
    • Select <Validate>, and pControl will check all completed fields or
    • Select <Cancel> to exit the view without saving any changes.
  5. Select whether this amendment is an originator or counterparty amendment, and enter any notes.

  6. A different user can then retrieve the amended Income / Expense transaction; review the changes and then either approve or reject.

    7. pControl will now hold a cash balance of $5001 for this account.

     

 Cancelling ‘Balance Difference' transactions

In this example, the initial bank balance for ‘Dollars and Sense Account 1’ was $10,000. The external bank account value is $6,500. A file is received from an external source into the U365: Cash Balances worktable showing the $6,500. When this file is loaded and validated, pControl will automatically write a balance difference journal transaction for the difference of $3,500.

Once investigated and an appropriate transaction created, the balance difference can be cancelled by:

  1. Select the required transaction from the ‘Maintain Income / Expense’ view
  2. Select <Delete / Cancel>

  3. Select whether this is an ‘Originator Cancel’ or ‘Counterparty cancelled’ transaction, and enter any processing notes as required.

    4. Note: The text shown in the ‘Processing Notes’ fields will be auto populated into this field. This can be modified if required.

  4. Select <Ok> to cancel the transaction, or <Cancel> to exit without completing the cancellation.

    5. The ‘cancel amend type’ of this transaction will now show as ‘Originator Cancellation’, and the status of the ‘Trade Capture and Validation’ status stream is showing ‘Trade Accepted’:

    Note: The view may need to be refreshed by selecting the binoculars icon to show the updated ‘Cancel Amend type’.
  5. Open the transaction in the properties view, by right clicking and selecting, ‘Properties’.
  6. Unlock the transaction by selecting the padlock icon
  7. Select <Ready to Auth> in the bottom right hand corner.

  8. Select a reason for cancellation and add any processing notes as required.

  9. Select <Ok> to cancel the transaction, or <Cancel> to exit without completing the cancellation.

    10. A second user can then review the cancellation, and approve or reject the cancellation.

  10. Second user selects Functions > Funds Processing > Capture and Maintain > Income / Expense.
  11. Select the cancelled transaction, and open the properties view.
  12. Unlock the transaction with the padlock icon.
    • Select <Authorise> to complete the cancellation, or <Reject> to exit the cancellation process.

Once the cancellation has been ‘Authorised’, the balance difference transaction is removed, leaving only the original cash balance of $10,000.

Where a bank balance file is received more than once per day, pControl will process each file in turn, and will calculate the difference each time a new file is received. The original balance difference transaction does not have to be cancelled manually.
