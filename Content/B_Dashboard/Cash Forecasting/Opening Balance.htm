Opening Balance

The opening balance column contains the sum of any financial events which meet all of the following selection criteria:

  • Internal state of the cash financial event is ‘Completed’
  • and the settlement date is less than the ‘From date’
  • and the financial event is not associated to a settlement instructions entity which is excluded from the Cash Forecast view. (Refer to Financial Events excluded from the Cash Forecast.)

Or

  • Where the financial event’s status code = ‘Balance Difference’ (indicating the balance calculated in pControl differs to the balance received from an external source).
    • The value of the balance difference transaction, (rather than financial event) is displayed in the opening balance column.
Note

All balance difference transactions will always display in the ‘Opening Balance’ column, regardless of the effective date of the balance difference transaction.
