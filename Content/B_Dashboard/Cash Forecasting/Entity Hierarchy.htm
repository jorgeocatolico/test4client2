Entity Hierarchy

The entity column contains a hierarchical drill down of entity descriptions for which an order transaction has been generated, and there are cash financial events associated with that order. The settlement date of those orders is generally in the future, meaning a positive or negative cash flow is anticipated on the settlement date.

Default Entity Hierarchy

The default hierarchy, shown below, is used in most scenarios, where an order is settled in either the same currency or a different currency to the Originator's currency. Where the order is settled in a different currency, pControl can convert the value back to the Originator's currency using the FX rates in the U178: FX Rates worktable.

 Row Descriptions for Entity Hierarchy

The following table describes each row type.

Row type Description

Client

Represents the legal entity the order is being executed for.

Originator

The originator of the ‘cash funding’ financial event.

Note: The term ‘Originator’ is also used to refer to the originator of an order transaction. When it refers to order transactions, the originator is the entity associated to a client requesting the order.

Alternate Funding

The alternate funding row will be displayed under all originators, regardless of whether there are any cash forecast values or not.

  • If there are no financial events there will be no further drill down available beyond this row.
  • If there are financial event/s associated to an asset which has been identified as an alternate funding source, via selection of the ‘Cash Forecasting Funding Source’ attribute, the next level of drill down options will be displayed, along with the corresponding cash forecast values.

For further detail, refer to Alternate Funding.

Other Party Code

Other party refers to the opposing party for the cash settlement. Where the transaction type is:

  • a transfer, the other party code refers to the other party involved in the transfer.
  • anything other than a transfer:
  • and the originator of the order is located above this current point in the hierarchy, the other party code is the 'counterparty'.
  • and the originator of the order is not located above this current point in the hierarchy, the other party code will be the 'originator'.

Cash Movement

Cash Movement is only displayed under ‘Alternate Funding’ when there is a financial event movement within the selected date range against an asset where the ‘Cash Forecasting Funding Source’ attribute is selected.

Failed settlements, and future settlements may also be displayed within the ‘cash movement’ row. The cash movement displayed here is from the ‘alternate parties’ perspective. Therefore, if the originator of the cash event is receiving funds, the value will be negative, and the ‘alternate balance’ will be reduced.
Bank Account Currency Code The three letter currency code of the cash financial event. Note: It is possible for one counterparty to have several associated settlement instruction entities, each with a different currency code. For further details refer to pControl User Guide – UOM.

Bank Account Code

This field will display the settlement instructions entity containing details of the bank account numbers, currency details, and methods of payment, which are associated to the originator on a financial event.

Cash Type

Cash type grouping indicates whether the cash financial event is linked to the originator of the transaction or not.

  • Where the financial event is linked to the originator of the transaction, the cash type is ‘Cash Funding’
  • Where the financial event is not linked to the originator of the transaction, the cash type is ‘Cash Requests’.

Status Code

Status code refers to the grouping that the internal pControl financial event status code is linked to. Valid group values include:

  • Pending – The state of the financial event is not in ‘Cancelled’, or ‘Completed’, and the settlement date of the financial event is greater than or equal to the ‘From date’.
  • Failed – Internal state not in ‘Cancelled’, or ‘Completed’, and the settlement date of the financial event is less than the ‘From Date’
  • Balance Difference – values are created when the bank account values imported into pControl are different to the value calculated by pControl. These are always displayed in the first column of the Cash Forecast table, regardless of the date range selected.
  • Settled – The state of the financial event is in ‘Completed’, and the settlement date of the financial event is greater than or equal to the ‘From Date’.

Note: the ‘state’ of cash financial events are displayed in processing mode on the Financial event view in the first column below:

Other Party Code

Other party refers to the opposing party for the cash settlement. Where the transaction type is:

  • a transfer, the other party code refers to the other party involved in the transfer.
  • anything other than a transfer,
  • and the originator of the order is located above this current point in the hierarchy, the other party code is the ‘counterparty’.
  • and the originator of the order is not located above this current point in the hierarchy, the other party code will be the ‘originator’.

Transaction Type Code

The code to indicate the type of transaction to which financial events are attached.

Some of the most common transaction types are ‘Sub’ ‘Red’, ‘Transfer’, and ‘Payments’.
Note

Entity descriptions are taken from pControl User Guide – Unitised Order Management. For further detail on entity types and order processing, refer to this User Guide.
