Altering Hierarchy

The cashforecast-config.xml file contains the entity hierarchy used in the cash forecasting web page. With the exception of ‘CLIENT’, all other levels of the hierarchy, i.e. those in the red box below can be reordered by a System Administrator.

To modify the entity hierarchy:

  1. Navigate to the .\etc folder
  2. Open the cashforecast-config.xml.

  3. Reorder any of the nodes in the red box
  4. Save and close the file
  5. Restart the container, and re-open the web page. (Clear browsing data if necessary).

 

Default Expansion Option

The Cash Forecasting page can be configured to open to a default expansion level if required. The value in the <mainBranch> row is used to determine the lowest level of expansion.

COUNTER_PARTY_CODE is the value selected as the ‘mainBranch’. Therefore, when the cash forecast page is opened, all levels of the hierarchy above COUNTER_PARTY_CODE will be expanded. This value can be replaced by a System Administrator with any other value from the <!—Possible Values row.
