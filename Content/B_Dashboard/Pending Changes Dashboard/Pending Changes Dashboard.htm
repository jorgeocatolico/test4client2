Pending Changes Dashboard

The Pending Changes Dashboard enables the quick review and navigation to events or objects that are pending authorisation. When dual authorisation functionality is activated, records that are added, amended or deleted, require a second person's authorisation before the record can be transitioned to a new state, for example, 'Ready to Authorise', or 'Authorised'.

The Pending Changes Dashboard provides a summary view of these events awaiting action and therefore directs the approver's attention to events with changes that require review. The Pending Changes Dashboard is configured in the System Administration module.

 Product Master Worktable Example

  1. Make a change to the MG_BALANCEDFUND record in the Product Master, changing the Strategy Format from Private to Public. The change in record is visible in red text.

  2. Once the record is saved, the authorisation status is updated and the original record viewable by selecting the expand/collapse icon.

    Transition the record using the ellipsis.

  3. The second user, or approver, will be able to see the change that is ready on the Pending Changes Dashboard. From here, the approver can select <Review> to navigate to the record in the Product Master worktable, where a transition can be made.

    Selecting a section of the donut, for example the green highlighted 'Ready to Authorise' section, navigates to the Product Master worktable where all 'Ready to Authorise' records with pending changes will be listed.

  4. Once the approver has transitioned the record to the next stage, the record will be removed from the Pending Changes Dashboard.

pControl Dashboard Configuration has been enhanced to support additional features to be used in pControl Web. Specifically, a dashboard can now be configured with date offsets and entity active types.

Date offsets allow the user to specify a large date offset both backwards and forwards in time. This means that a dashboard can display a large range of time based data, while primary date filters are still in the context of the current date. Entity type filtering also allows a dashboard to show data for Active entities, Inactive entities or both. This functionality will allow dashboards to be configured in such a way as to provide the maximum value to users, without having to change the date ranges and filters within pControl Web. An example configuration that is suggested to capture an Attention Status Monitor range of data would be:

  • From Date Offset: From Date - 365
  • To Date Offset: Now +31
  • Active Status: All
